第 14 章

Vision Transformer

2021 年，Dosovitsky 等人在论文《An Image is Worth 16×16 Words》中引入了将 Transformer 用于计算机视觉的想法。与卷积网络相比，Vision Transformer 能取得出色的结果，且需要更少的训练资源。

核心思想：图像即序列

ViT 的关键创新：将图像切分为固定大小的补丁（Patch），将每个补丁视为一个"词"，然后用标准 Transformer 编码器处理。

补丁嵌入（Patch Embedding）

class PatchEmbedding(nn.Module):
    def __init__(self, d_model, img_size, patch_size, n_channels):
        super().__init__()
        self.linear_project = nn.Conv2d(
            n_channels,      # 输入通道
            d_model,         # 输出维度
            kernel_size=patch_size,
            stride=patch_size
        )

    def forward(self, x):
        # (B, C, H, W) → (B, d_model, P_col, P_row) → (B, P, d_model)
        x = self.linear_project(x)
        x = x.flatten(2)
        x = x.transpose(1, 2)
        return x

类别 Token 与位置编码

ViT 在序列开头添加一个可学习的类别 token（[CLS]），用于聚合全局信息做分类。同时添加位置编码让模型知道每个补丁的空间位置。

Transformer 编码器

与 GPT-2 的 Decoder-Only 不同，ViT 使用Transformer 编码器（双向注意力）。每个补丁可以看到所有其他补丁。

ViT vs CNN

CNN 通过局部卷积提取特征，ViT 通过全局自注意力提取特征。ViT 的优势在于能捕捉长距离依赖，但需要更多数据才能超越 CNN。

第 15 章

CLIP — 图文对比学习

CLIP 是一场革命：它允许通过"预测哪些图像和文本配对在一起"来识别开放世界中的对象，不再局限于固定类别。

核心原理：对比学习

CLIP 同时训练图像编码器和文本编码器，使匹配的图文对在嵌入空间中距离更近，不匹配的距离更远。

# CLIP 对比学习（简化）
# batch 中 N 个图文对
image_features = image_encoder(images)    # (N, D)
text_features  = text_encoder(texts)      # (N, D)

# 计算相似度矩阵 (N, N)
logits = image_features @ text_features.T / temperature

# 对角线是正样本，其余是负样本
labels = torch.arange(N)
loss = cross_entropy(logits, labels) + cross_entropy(logits.T, labels)

图像编码器与文本编码器

CLIP 的图像编码器使用 Vision Transformer（或 ResNet），文本编码器使用 Transformer。两者将各自的输入映射到同一嵌入空间。

零样本分类

CLIP 最强大的能力：无需训练即可分类。将类别名称构造为文本（"a photo of a cat"），计算图像与每个文本的相似度，取最大的即为分类结果。

CLIP 的枢纽地位

CLIP 是多模态 AI 的关键枢纽：在 ClipCap 中驱动图生文，在 DALL-E2 中提供图文嵌入空间。理解 CLIP 是理解后续多模态模型的基础。

第 16 章

ClipCap — 图生文

工作原理

ClipCap 结合 CLIP 和 GPT-2 实现图像标题生成：

# ClipCap 工作流程
输入图片 → CLIP → 图像嵌入
                        ↓
               Mapping Network (MLP)
                        ↓
               前缀嵌入 (Prefix Embeddings)
                        ↓
              GPT-2 → 逐 token 生成标题
                        ↓
              "A drawing of a blue car"

关键问题：嵌入空间不匹配

CLIP 和 GPT-2 的嵌入空间不同，不能直接连接。ClipCap 使用映射网络（Mapping Network）将 CLIP 嵌入映射到 GPT-2 的嵌入空间，生成"前缀嵌入"。

两种变体

  • 微调 GPT-2：用 MLP 作为映射网络，GPT-2 和 MLP 都训练
  • 不微调 GPT-2：用 Transformer 作为映射网络，只训练映射网络

推理过程

逐 token 自回归生成：前缀嵌入 → 预测第一个 token → 前缀+第一个token → 预测第二个 → ... → 生成 <EOS> 或达到最大长度。

第 17-18 章

扩散模型 — 从原理到数学

扩散模型是近年来 AI 生成艺术突破的主要因素，是 DALL-E 和 Midjourney 等知名模型的核心机制。

通俗理解

想象一杯透明的水加入黄色液体——液体逐渐扩散，水不再透明。这是正向扩散。反向操作（将混合物恢复原状）则困难得多——这是反向扩散。

在图像中：正向扩散 = 逐步添加噪声直到变成纯噪声；反向扩散 = 从纯噪声逐步去噪恢复图像。

正向扩散过程

逐步向图像添加高斯噪声。经过 T 步后，图像变成纯噪声。利用重参数化技巧，可以直接采样任意时间步 t 的噪声图像：

xₜ = √(αₜ) · x₀ + √(1 - αₜ) · ε

其中 ε ~ N(0, I), αₜ = ∏(1 - βᵢ)

完整的闭式公式推导（重参数化技巧、递归展开、方差合并）见正向扩散闭式公式。

反向扩散过程

训练神经网络预测每一步添加的噪声，然后从噪声图像中减去预测的噪声：

# 扩散模型训练（简化）
for each training step:
    x₀ = sample_real_image()           # 真实图像
    t = random_timestep()              # 随机时间步
    ε = sample_noise()                 # 随机噪声
    xₜ = add_noise(x₀, t, ε)          # 加噪
    predicted_noise = model(xₜ, t)     # 预测噪声
    loss = mse(ε, predicted_noise)     # MSE 损失

U-Net 神经网络

扩散模型使用 U-Net 作为去噪网络。U-Net 有编码器（下采样）和解码器（上采样）路径，中间有跳跃连接，保留细节信息。

数学理论：郎之万动力学

扩散模型的理论基础来自郎之万动力学——一种 MCMC 采样方法。在梯度下降的每一步添加噪声项，使采样过程能探索整个分布：

xₜ₊₁ = xₜ - (ε²/2)∇f(xₜ) + √ε · N(0, I)

从概率分布的指数族形式到梯度下降再到加噪采样的完整推导，见郎之万动力学。

预测噪声 vs 预测图像

虽然扩散模型可以预测"上一步的图像"或"添加的噪声"，但预测噪声更简单、更有效。这是 DDPM 的关键设计选择。

第 19 章

条件扩散模型

为什么要条件化？

普通扩散模型只能随机生成图像。我们希望能控制生成内容——给定标签、文本或低分辨率图像，生成对应的图像。

添加条件的方法

将条件 y 添加到神经网络的输入中，把建模 p(x) 转为建模 p(x|y)。对于标签条件，通过嵌入层将标签转为向量，与时间步嵌入相加：

pθ(x₀|y) = ∫ pθ(x₀|x₁, y) · pθ(x₁|x₂, y) · ... · pθ(x_{T−1}|x_T, y) · p(x_T) dx₁...dx_T

从无条件到条件的完整推导、条件嵌入方式、分类器指引 vs 无分类器指引的对比，见条件扩散p(x|y)推导。

# 条件扩散模型
# 噪声预测网络接收额外条件 y
ε_θ(xₜ, t, y)  # y 可以是标签、文本等

# 标签条件：嵌入层 + 时间步嵌入
label_emb = embedding(y)       # 标签 → 向量
time_emb = sinusoidal(t)       # 时间步 → 向量
condition = label_emb + time_emb

分类器指引（Classifier Guidance）

在反向扩散的每一步，使用分类器的梯度引导生成方向，使生成的图像更符合条件。

无分类器指引（Classifier-Free Guidance）

不需要额外分类器。同时训练有条件和无条件模型，在推理时通过两者的差值引导生成。这是 Stable Diffusion 使用的方法。

Stable Diffusion

Stable Diffusion 是条件扩散模型的里程碑：在潜在空间（而非像素空间）进行扩散，大幅降低计算成本，使得在消费级 GPU 上生成高质量图像成为可能。

第 20 章

DALL-E2 — 文生图完整实现

DALL-E2 是条件扩散模型的应用巅峰：输入文本提示词，输出图像。它使用 unCLIP 方法，需要训练三个模型。

三个模型的架构

# DALL-E2 = CLIP + 先验模型 + 条件扩散模型
1. CLIP 模型
   → 学习图文对齐的嵌入空间
   → 训练完毕后冻结

2. 先验模型 (Prior Model)
   → Decoder-Only Transformer
   → 输入: 文本标题 + CLIP文本嵌入 + 时间步 + 噪声CLIP图像嵌入
   → 输出: CLIP图像嵌入
   → 作用: 根据文本预测高质量的图像条件

3. 条件扩散模型 (Decoder)
   → UNet 架构
   → 输入: CLIP图像嵌入 + 噪声图像 + 时间步
   → 输出: 去噪后的图像
   → 作用: 根据条件生成最终图像

先验模型的训练

先验模型的核心任务：根据文本标题预测 CLIP 图像嵌入。它是一个 Decoder-Only Transformer，接收 6 个输入：

  • 文本标题
  • CLIP 文本嵌入（冻结 CLIP 生成）
  • 时间步嵌入
  • 加噪声的 CLIP 图像嵌入
  • 可学习嵌入
  • 因果注意力掩码
关键注意

先验模型是对 CLIP 图像嵌入添加噪声进行扩散，而不是对图像本身！这是一个容易混淆的点。

条件扩散模型（Decoder）

Decoder 使用 UNet 架构，包含残差块、注意力块、下采样和上采样。以先验模型生成的 CLIP 图像嵌入作为条件，从纯噪声逐步去噪生成图像。

完整生成流程

# DALL-E2 文生图流程
输入: "A drawing of a blue car"

Step 1: 文本 → 冻结的 CLIP → CLIP 文本嵌入
Step 2: CLIP文本嵌入 → 先验模型(扩散) → CLIP图像嵌入
Step 3: CLIP图像嵌入 + 纯噪声 → 条件扩散模型(去噪) → 生成图像

输出: 🖼️ (一辆蓝色汽车的图像)

训练配置与结果

本章提供了完整的训练配置，包括数据集准备、模型超参数、训练循环和损失计算。训练完成后能生成与文本描述匹配的图像。

DALL-E2 的精髓

DALL-E2 的精髓在于两阶段架构：先用先验模型将文本"翻译"为图像条件（CLIP 图像嵌入），再用扩散模型根据条件生成图像。这种解耦设计让每个子模型都能专注做好自己的任务。
