LLM

交叉熵损失函数推导

从极大似然估计出发，推导 LLM 训练使用的交叉熵损失。

第 1 步：极大似然估计

假设训练数据为 "abcd"，我们希望模型对每个位置预测正确的下一个 token 的概率越大越好。所有位置的联合概率为：

Pθ(d|abc) · Pθ(c|ab) · Pθ(b|a)

第 2 步：取对数（log 似然）

概率连乘难优化且数值下溢，取 log 转为连加：

log{ Pθ(d|abc) · Pθ(c|ab) · Pθ(b|a) }
= log Pθ(d|abc) + log Pθ(c|ab) + log Pθ(b|a)

第 3 步：最大化 ↔ 最小化负对数

最大化 log 似然等价于最小化负 log 似然：

minimize: −log Pθ(d|abc) − log Pθ(c|ab) − log Pθ(b|a)

第 4 步：识别为交叉熵

这正是交叉熵损失。给定输入 "ab"，目标分类是 "c"，模型输出 c 的概率为 p，则损失为 −log p。

更一般地，对真实分布 P（one-hot）和预测分布 Q，交叉熵为：

H(P, Q) = −Σₓ P(x) log Q(x)
直观理解

LLM 本质是个分类模型：类别 = 词表中的所有 token（可能数万个）。每次预测就是一次"分类"，正确 token 的概率越大，损失越小。

LLM

自注意力机制（Self-Attention）

核心公式

Attention(Q, K, V) = softmax(QKᵀ / √dₖ) · V

三个矩阵的来源

输入序列 X（每个 token 的嵌入向量）通过三个不同的线性变换得到 Q、K、V：

Q = X · WQ    # Query：当前 token 想查询什么
K = X · WK    # Key：每个 token 提供的"标签"
V = X · WV    # Value：每个 token 携带的实际信息

逐步分解

1. QKᵀ：计算每个 token 与所有 token 的相似度（点积）。点积越大，两个 token 越相关。

2. /√dₖ：缩放因子，防止点积过大导致 softmax 梯度消失。dₖ 是 Key 的维度。

3. softmax：把相似度归一化为概率分布（权重和为 1）。

4. ·V：用注意力权重对所有 token 的 Value 加权求和，得到当前 token 的新表示。

多头注意力

将嵌入空间分成 h 个子空间，每个子空间独立计算注意力，最后拼接。这让模型能同时关注不同类型的关系（语法、语义、共指等）。

Decoder-Only 的关键约束

GPT 系列使用因果掩码（causal mask）：每个 token 只能"看到"它之前的 token，看不到未来。这保证了自回归生成的正确性。

LLM

预测下一个 Token（NTP）

大语言模型要解决的核心问题是预测下一个 token（Next Token Prediction, NTP）。

训练数据的构造

原始文本 "君不见黄河之水天上来" 被程序切分为输入-目标对：

输入              预测目标
君不见黄河        → 之
君不见黄河之      → 水
君不见黄河之水    → 天
君不见黄河之水天  → 上

预测目标不是人工标注，而是程序通过切分文本自动生成——所以叫自监督学习。

滑动窗口

为了利用 GPU 并行，训练时使用固定长度的滑动窗口（context length），每个位置同时预测下一个 token。

为什么这个简单目标如此强大？

预测下一个 token 需要理解语法、语义、世界知识、推理能力。一个能完美预测下一个 token 的模型，几乎能完成任何语言任务。

RL

贝尔曼方程

贝尔曼方程是强化学习的核心递推关系，将价值函数分解为"即时奖励 + 下一状态的价值"。

状态价值函数的贝尔曼方程

V(s) = E[ Rₜ + γ · V(Sₜ₊₁) | Sₜ = s ]

推导思路

价值函数 V(s) 定义为从状态 s 出发的期望累积回报：

V(s) = E[ Gₜ | Sₜ = s ]

其中 Gₜ = Rₜ + γ·Rₜ₊₁ + γ²·Rₜ₊₂ + ...

将 Gₜ 拆分为即时奖励 Rₜ 和折扣后的未来回报 γ·Gₜ₊₁：

Gₜ = Rₜ + γ · Gₜ₊₁

代入期望：

V(s) = E[ Rₜ + γ · Gₜ₊₁ | Sₜ = s ]
= E[ Rₜ | Sₜ = s ] + γ · E[ Gₜ₊₁ | Sₜ = s ]
= E[ Rₜ + γ · V(Sₜ₊₁) | Sₜ = s ]

动作价值函数 Q(s, a) 的贝尔曼方程

Q(s, a) = E[ Rₜ + γ · E_{a'~π}[ Q(Sₜ₊₁, a') ] | Sₜ = s, Aₜ = a ]
为什么贝尔曼方程重要？

它将"长期累积回报"转化为"即时奖励 + 下一状态价值"的递推形式，使我们可以用迭代或动态规划求解价值函数。TD 学习、Q-learning、Actor-Critic 都建立在贝尔曼方程之上。

RL

策略梯度定理

目标函数

策略 πθ 的目标：最大化期望回报

J(θ) = E_{τ~πθ}[ G(τ) ]

其中轨迹 τ = (S₀, A₀, R₀, S₁, A₁, R₁, ..., S_{T+1})，回报 G(τ) = R₀ + γR₁ + γ²R₂ + ... + γᵀRᵀ。

轨迹的概率

Pr(τ) = p(S₀) · ∏ₜ πθ(Aₜ|Sₜ) · p(Sₜ₊₁|Sₜ, Aₜ)

求梯度

对 J(θ) 求梯度，关键是应用log-导数技巧：

∇θ log Pr(τ) = ∇θ Σₜ log πθ(Aₜ|Sₜ)

(因为只有 πθ 含 θ，p(S₀) 和 p(Sₜ₊₁|Sₜ,Aₜ) 与 θ 无关)

由此得到策略梯度定理：

∇θ J(θ) = E_{τ~πθ}[ Σₜ G(τ) · ∇θ log πθ(Aₜ|Sₜ) ]

直观解读

  • G(τ)：轨迹的回报，作为该轨迹的"权重"
  • ∇θ log πθ(Aₜ|Sₜ)：增大在状态 Sₜ 下采取动作 Aₜ 的对数概率的梯度
  • 整体含义：高回报轨迹上的动作被强化，低回报轨迹上的动作被削弱
期望无法精确计算

真实期望需要对所有轨迹求平均，但轨迹数量是天文数字（如 2²⁰⁰）。实际使用蒙特卡洛采样：采 n 条轨迹，用样本平均近似期望。

RL

广义优势估计（GAE）

问题背景

优势函数 A(s, a) = Q(s, a) − V(s) 衡量动作 a 相对平均水平的优劣。估计 A 有两种极端：

  • 1 步 TD：偏差小，方差大（Aₜ ≈ δₜ = Rₜ + γV(Sₜ₊₁) − V(Sₜ)）
  • 蒙特卡洛：偏差大，方差小（Aₜ ≈ Gₜ − V(Sₜ)）

GAE 公式

GAE 通过多步 TD 误差的指数加权平均，在偏差和方差之间取得平衡：

Aₜ^GAE(γ, λ) = Σₗ₌₀^∞ (γλ)ˡ · δₜ₊ₗ

其中 δₜ = Rₜ + γV(Sₜ₊₁) − V(Sₜ) 是 TD 误差

参数 λ 的作用

  • λ = 0：Aₜ^GAE = δₜ，退化为 1 步 TD（偏差大、方差小）
  • λ = 1：Aₜ^GAE = Gₜ − V(Sₜ)，退化为蒙特卡洛（偏差小、方差大）
  • 0 < λ < 1：在两者之间权衡（典型值 0.95）

展开形式

Aₜ^GAE = δₜ + γλ·δₜ₊₁ + (γλ)²·δₜ₊₂ + ...

每多加一项，就多考虑一步未来，但权重以 γλ 衰减——既不完全相信远处的价值估计，也不完全忽视。

RL

REINFORCE 推导

原始策略梯度的问题

原始策略梯度中，每个时刻 t 的动作权重都是 G(τ)（整条轨迹的回报）。但时刻 t 之前的奖励与时刻 t 的动作好坏无关——它们是"噪声"。

REINFORCE 的改进

用 Gₜ（时刻 t 之后的累积回报）替代 G(τ)：

Gₜ = Rₜ + γRₜ₊₁ + γ²Rₜ₊₂ + ... + γ^{T−t}R_T

REINFORCE 版策略梯度定理：

∇θ J(θ) = E_{τ~πθ}[ Σₜ Gₜ · ∇θ log πθ(Aₜ|Sₜ) ]

为什么这样改进？

采取动作 Aₜ 的好坏应该由动作之后的回报来评估。之前的奖励已经发生，与当前动作无关。去除这部分噪声能降低梯度估计的方差。

REINFORCE 名字的由来

REward Increment = Nonnegative Factor × Offset Reinforcement × Characteristic Eligibility（奖励增量 = 非负因子 × 偏移强化 × 特征资格）的首字母缩写。

RLHF

KL 散度（Kullback-Leibler Divergence）

定义

KL 散度衡量两个概率分布 p 和 q 的差异：

连续型：D_KL(p‖q) = ∫ p(x) log[ p(x)/q(x) ] dx

离散型：D_KL(p‖q) = Σₓ p(x) log[ p(x)/q(x) ]

三大性质

  • 非负性：D_KL(p‖q) ≥ 0，当且仅当 p = q 时为 0
  • 非对称性：D_KL(p‖q) ≠ D_KL(q‖p)
  • 差异单调性：两分布差异越大，KL 散度越大

抛硬币示例

真实分布 p：正面 70%，反面 30%。

估计 1（50/50）：D_KL = 0.7·log(0.7/0.5) + 0.3·log(0.3/0.5) ≈ 0.082

估计 2（20/80）：D_KL = 0.7·log(0.7/0.2) + 0.3·log(0.3/0.8) ≈ 0.58

估计 3（70/30，正确）：D_KL = 0.7·log(1) + 0.3·log(1) = 0

在 RLHF 中的关键作用

KP 散度作为惩罚项，防止微调后的 LLM πθ 偏离参考模型 πref 太远：

β · D_KL[ πθ ‖ πref ]
为什么必须有 KL 惩罚？

没有 KL 约束，RL 模型会奖励黑客（Reward Hacking）——利用奖励模型的漏洞，生成人类读不通但 RM 误判高分的文本（如重复乱码）。KL 惩罚拽住模型，让它仍然是"人话"。

与信息论的关系

KL 散度 = 交叉熵 − 熵：D_KL(p‖q) = H(p, q) − H(p)

当 p 固定时，最小化交叉熵等价于最小化 KL 散度。

RLHF

重要性采样（Importance Sampling）

核心问题

想计算 E_{x~p}[f(x)]，但无法从 p 采样，只能从 q 采样。怎么办？

解决方法

E_{x~p}[f(x)] = E_{x~q}[ (p(x)/q(x)) · f(x) ]

比值 p(x)/q(x) 称为重要性权重。它修正了"用 q 的样本估计 p 的期望"带来的偏差。

方差问题

重要性采样的方差为：

Var(μ̂_q) = (1/N) · [ E_{x~p}[ (p(x)/q(x)) · f(x)² ] − (E_{x~p}[f(x)])² ]
方差的隐患

如果 p(x)/q(x) 在某些位置很大，方差会爆炸。所以 p 和 q 必须足够接近，否则估计不可靠。

在 PPO 中的应用

PPO 用旧策略 πθ_old 采样的数据更新新策略 πθ，通过重要性权重修正：

ρ = πθ(a|s) / πθ_old(a|s)

PPO 用 clip 操作限制 ρ 在 [1−ε, 1+ε] 范围内，避免方差爆炸，这就是替代目标的核心。

RLHF

PPO 替代目标推导

起点：策略改进目标

希望保证新策略不比旧策略差：J(θ) − J(θ_old) ≥ 0

第 1 步：差值展开

经过推导（利用 A^{πθ_old} 的定义和价值函数性质）：

J(θ) − J(θ_old) = E_{τ~πθ}[ Σₜ γᵗ · A^{πθ_old}_t ]

第 2 步：用重要性采样改写

问题：上式期望需要用新策略 πθ 采样，但更新前新策略还不存在。用重要性采样转为用旧策略 πθ_old 采样的数据：

J(θ) − J(θ_old) ≈ E_{τ~πθ_old}[ Σₜ (πθ(Aₜ|Sₜ)/πθ_old(Aₜ|Sₜ)) · A^{πθ_old}_t ]
= J^CPI(θ)

这就是 CPI（Conservative Policy Iteration）目标。

第 3 步：clip 防止方差爆炸

重要性权重 ρ = πθ/πθ_old 可能很大，导致方差爆炸。PPO 用 clip 限制：

J^PPO(θ) = E[ min( ρ·A, clip(ρ, 1−ε, 1+ε)·A ) ]

其中 ρ = πθ(a|s) / πθ_old(a|s)

min 操作的作用

取 min 是悲观策略：

  • 当 A > 0（好动作）：clip 限制 ρ ≤ 1+ε，防止单次过度强化
  • 当 A < 0（坏动作）：clip 限制 ρ ≥ 1−ε，防止单次过度削弱
PPO 的精髓

PPO 用一个简单的 clip 操作，既保留了重要性采样的样本效率，又避免了方差爆炸。它是"足够好"和"足够简单"的平衡——这也是它被广泛使用的原因。

RLHF

GRPO 组相对优势

核心创新

GRPO 不需要训练 Critic 网络估计价值，而是用一组轨迹的相对优势替代。

计算流程

对每个问题 qᵢ，采样 G 条回答 τᵢ,₁, ..., τᵢ,G，计算每条的奖励 rᵢ,ⱼ。然后组内归一化：

μᵢ = mean(rᵢ,₁, ..., rᵢ,G)
σᵢ = std(rᵢ,₁, ..., rᵢ,G)

Aᵢ,ⱼ = (rᵢ,ⱼ − μᵢ) / σᵢ

直观解读

  • 表现高于平均的回答 → 正优势 → 被强化
  • 表现低于平均的回答 → 负优势 → 被削弱
  • 不需要训练额外的价值网络，节省 GPU 内存

GRPO 目标函数

J^GRPO(θ) = (1/G) Σᵢ (1/|τᵢ|) Σₜ min[ρ·A, clip(ρ, 1−ε, 1+ε)·A] − β·D_KL[πθ‖πref]

DAPO 的改进

字节提出的 DAPO 是 GRPO 的变种：

  • Token 级损失：每个 token 的优势 = 其所在轨迹的组相对优势
  • 移除 KL 散度：不需要参考策略网络，进一步节省内存
为什么 GRPO 适合推理任务？

推理任务（数学、代码）有明确的正确性标准，适合作为奖励信号。GRPO 的"多条采样取最优"天然适合——对同一问题生成多个回答，好的强化、差的削弱。DeepSeek-R1 正是用此实现推理突破。

RLHF

DPO 目标函数推导

核心思想

DPO（Direct Preference Optimization）的洞察：你的 LLM 本身就是奖励模型。不需要单独训练 RM，直接从偏好数据优化 LLM。

偏好数据

{
  "prompt": "这部电影怎么样？",
  "chosen": "这部电影很好看。",      # 偏好
  "rejected": "这部电影不好看。"     # 厌恶
}

DPO 目标函数

J_DPO(θ; πref) = E[ log σ( β · ( log[πθ(yw|x)/πθ(yl|x)] − log[πref(yw|x)/πref(yl|x)] ) ) ]

推导关键步骤

第 1 步：补全概率

给定 prompt x，模型输出 y 的概率：

πθ(y|x) = ∏ₜ P_LLM(yₜ | x, y<ₜ)

第 2 步：log 商 = log 差

利用 log(A/B) = log A − log B：

log[πθ(yw|x)/πθ(yl|x)] − log[πref(yw|x)/πref(yl|x)]
= log[πθ(yw|x)/πref(yw|x)] − log[πθ(yl|x)/πref(yl|x)]

第 3 步：训练动态

  • 训练开始：πθ = πref，内项为 0，σ(0) = 0.5
  • 训练进行：πθ(yw|x)/πref(yw|x) 增大（偏好回答概率上升）
  • πθ(yl|x)/πref(yl|x) 减小（厌恶回答概率下降）

DPO 的三个问题

DPO 的局限性

1. 分子分母同时变化：可能 chosen 和 rejected 的概率都上升，只是 chosen 上升更多，模型仍可能输出厌恶回答。

2. 容易过拟合：随着训练 πθ(yw)→1, πθ(yl)→0，正则化项失效，模型失去探索能力。DPO 训练轮数不能多。

3. 偏好循环：数据中 A>B 和 B>A 同时存在时，模型无法学习。

DPO 是强化学习吗？

本书作者认为 DPO 不算严格意义的强化学习——它需要预定的偏好数据集，而真正的 RL 是策略自己采样数据。但 DPO 仍是有效的对齐方法。

RLHF

即时奖励 Rₜ 的计算（InstructGPT）

问题背景

在倒立摆环境中，每步奖励就是 +1。但在 LLM 中，奖励模型（RM）只能给完整补全打分，不能给单个 token 打分。如何把"序列级奖励"转化为"token 级奖励"？

InstructGPT 的精妙设计

分两种情况计算每个 token 的即时奖励：

情况 1：<eos> 之前的 token

Rₜ = −β · log[ πθ(yₜ|x, y<ₜ) / πref(yₜ|x, y<ₜ) ]

这部分只有 KL 惩罚，没有 RM 分数——因为 RM 还没看到完整序列。

情况 2：<eos>（最后一个 token）

R_T = RM 分数 − β · log[ πθ(y_T|x, y

此时序列结束，RM 给出整体评分，作为最后一个 token 的奖励。

KL 惩罚的作用

关键秘诀

KL 惩罚项 β·log[πθ/πref] 是 InstructGPT 成功的关键。它逐 token 计算——如果 πθ 对某 token 的概率远高于 πref，说明模型在"冒险"偏离原轨道，惩罚变大。

RM 分数推模型往"人类喜欢"的方向走；KL 惩罚拽住模型不让它跑太偏。两者平衡才能产生有用的对齐。

稀疏奖励的处理：GAE

RM 分数只在序列末尾出现（稀疏奖励）。靠 GAE 把最后的奖励"抹匀"分摊给前面的每个 token，让前面的词也能获得梯度更新。

多模态

正向扩散闭式公式推导

目标

推导任意时间步 t 的噪声图像 xₜ 的直接采样公式，避免迭代加噪。

前置：重参数化技巧

如果 z ~ N(μ, σ²)，那么 z = μ + σ·ε，其中 ε ~ N(0, 1)。

两个独立正态分布的和仍是正态分布：X ~ N(μ_X, σ²_X), Y ~ N(μ_Y, σ²_Y) → X+Y ~ N(μ_X+μ_Y, σ²_X+σ²_Y)。

单步加噪公式

xₜ = √(1−βₜ) · xₜ₋₁ + √βₜ · εₜ₋₁

递归展开

记 αₜ = 1 − βₜ， ᾱₜ = ∏ᵢ₌₁ᵗ αᵢ。逐步展开：

xₜ = √αₜ · xₜ₋₁ + √(1−αₜ) · εₜ₋₁
= √αₜ · (√αₜ₋₁ · xₜ₋₂ + √(1−αₜ₋₁) · εₜ₋₂) + √(1−αₜ) · εₜ₋₁
= √(αₜ·αₜ₋₁) · xₜ₋₂ + √(αₜ(1−αₜ₋₁)) · εₜ₋₂ + √(1−αₜ) · εₜ₋₁

两个独立高斯噪声项合并（方差相加）：

N(0, αₜ(1−αₜ₋₁)I) + N(0, (1−αₜ)I) = N(0, (1−αₜ·αₜ₋₁)I)

继续递归：

= √(αₜ·αₜ₋₁) · xₜ₋₂ + √(1−αₜ·αₜ₋₁) · ε
= ... ...
= √(αₜ·αₜ₋₁·...·α₁) · x₀ + √(1−αₜ·αₜ₋₁·...·α₁) · ε

最终闭式公式

xₜ = √ᾱₜ · x₀ + √(1−ᾱₜ) · ε

其中 ε ~ N(0, I), ᾱₜ = ∏ᵢ₌₁ᵗ (1−βᵢ)
为什么这个公式重要？

它让我们可以在任意时间步 t 直接采样 xₜ，不需要迭代 t 次。训练时随机采样 t，直接得到 (x₀, xₜ) 对，大幅加速训练。

多模态

郎之万动力学（Langevin Dynamics）

背景

扩散模型的理论基础来自郎之万动力学——一种 MCMC（马尔可夫蒙特卡洛）采样方法。

概率分布的指数族形式

假设目标分布 p(x) 属于指数分布族：

p(x) ∝ e^{−f(x)} ⟹ log p(x) = −f(x) + 常数

梯度下降求模态

如果我们只知道 f(x) 的梯度 ∇f(x)，可以用梯度下降找分布的峰值（模态）：

xₜ = xₜ₋₁ − ηₜ · ∇f(xₜ₋₁) ... (2)

峰值对应训练数据中"好"图像集中的区域——去到峰值就能得到好图像。

加入噪声项：郎之万采样

纯梯度下降只能找到一个峰值。郎之万采样在每个梯度步添加噪声项：

xₜ₊₁ = xₜ − (ε²/2)·∇f(xₜ) + √ε · N(0, I) ... (3)

噪声让采样过程能"跳出"局部峰值，探索整个分布，最终收敛到目标分布 p(x)。

稳定性问题

如果想加快采样，可能选择很大的 ε。但 ε 太大时方程 (3) 会不稳定并发散。扩散模型通过多步小噪声的方式保证收敛——这就是为什么扩散模型需要数百步迭代。

与扩散模型的关系

扩散模型的反向过程可以看作离散化的郎之万动力学：每步用神经网络预测 ∇f(xₜ)（即得分函数），然后用小步长去噪。这就是得分匹配视角下的扩散模型。

多模态

条件扩散模型 p(x|y) 推导

从无条件到条件

普通扩散模型建模 p(x)，无法控制生成内容。我们希望建模 p(x|y)，通过条件 y（标签、文本、低分辨率图像等）控制生成。

第 1 步：无条件模型的边际化

pθ(x₀) = ∫ pθ(x₀, x₁, ..., x_T) dx₁...dx_T
= ∫ pθ(x₀|x₁) · pθ(x₁|x₂) · ... · pθ(x_{T−1}|x_T) · p(x_T) dx₁...dx_T

第 2 步：在每一步添加条件 y

最简单的方法是在每个转移概率中加入条件 y：

pθ(x₀|y) = ∫ pθ(x₀|x₁, y) · pθ(x₁|x₂, y) · ... · pθ(x_{T−1}|x_T, y) · p(x_T) dx₁...dx_T

第 3 步：神经网络接收 y

每步的去噪网络从 εθ(xₜ, t) 变为 εθ(xₜ, t, y)：

pθ(xₜ₋₁|xₜ, y) = N(xₜ₋₁; μθ(xₜ, t, y), σ²_q(t)·I)

条件的嵌入

  • 标签 y：通过嵌入层转为向量，与时间步嵌入相加
  • 文本 y：通过 CLIP 文本编码器转为向量
  • 图像 y：通过编码器转为特征图

分类器指引 vs 无分类器指引

分类器指引：在反向扩散每步，用分类器 ∇log p(y|xₜ) 引导生成方向。

无分类器指引（CFG）：同时训练有条件 εθ(xₜ, t, y) 和无条件 εθ(xₜ, t)，推理时用两者的差值引导。Stable Diffusion 使用此方法。

DALL-E2 的条件

DALL-E2 的 Decoder 用先验模型生成的 CLIP 图像嵌入作为条件 y。先验模型把"文本"翻译成"图像条件"，这是 DALL-E2 两阶段架构的精髓。
