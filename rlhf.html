PART III

人类反馈强化学习

将强化学习应用于大语言模型：从预训练到 SFT，再到 DPO、PPO、GRPO 三种对齐方法。本章是 LLM 与 RL 两条主线的交汇点。

第 10 章

LLM 训练流程概述

三阶段训练流程

大语言模型的完整训练流程分为三个阶段：

# LLM 训练三阶段
Stage 1: 预训练 (Pre-training)
  → 自回归预测下一个 token
  → 学习语言的通用统计规律

Stage 2: 监督微调 (SFT)
  → 给定指令，预测回答的下一个 token
  → 让模型遵循指令格式

Stage 3: RLHF (基于人类反馈的强化学习)
  → 用 RL 优化模型使其符合人类偏好
  → DPO / PPO / GRPO 三种方法

预训练：自监督学习

训练数据 "abcd"，目标函数：

J(θ) = πθ(b|a) · πθ(c|a,b) · πθ(d|a,b,c)

最大化正确预测下一个 token 的概率

监督微调（SFT）

SFT 也是"预测下一个词"，但只在回答部分计算损失：

输入:  How are you? I am fine.
                    ↓↓↓↓↓↓↓↓
目标:  are you? I am fine. 
       [指令部分不计算损失]  [回答部分计算损失]
危险：灾难性遗忘

SFT 一般不会训练很多轮！如果训练过多，模型会过拟合微调数据，发生灾难性遗忘——丢失预训练阶段学到的通用能力。因为 SFT 目标函数中没有约束新旧模型的偏差（KL 散度）。

RLHF：为什么需要强化学习？

当我们谈论 LLM 的强化学习时，发生了根本性的转变：

  • 优化目标是人类偏好，而不是对环境的掌控
  • 数据来自人类的判断，而不是环境互动
  • 需要在奖励最大化与保持接近原始行为之间取得平衡

在 LLM 这个 RL 环境中：

# LLM 即 RL 环境
策略 πθ    = LLM
状态 Sₜ    = 上下文（已生成的文本）
动作 Aₜ    = 输出一个 token
动作概率   = πθ(token | context)
奖励 Rₜ    = 奖励模型的评分 - KL散度惩罚
机器学习类比 RLHF = 给 LLM 装上"判分老师"的微调
SFT 监督微调 给 LLM 看示范答案，模仿回答
↔
监督学习 / 微调 给模型标注数据，学习输入-输出映射
RLHF 强化学习 不告诉标准答案，只给打分，自己探索
↔
主动学习 / 弱监督 没有精确标签，只有反馈信号
类比要点：SFT 像传统的监督微调——有人工标注的标准答案。RLHF 像弱监督学习——没有标准答案，只有"这个回答好不好"的评分。RLHF 的价值在于：人类评判"好不好"比写"标准答案"容易得多，所以能大规模收集反馈数据。这就是 OpenAI 所说的"如何用数学定义'乐于助人'"——用人类偏好数据训练奖励模型，再用 RL 优化。

三种 RLHF 方法

方法一

DPO — 直接偏好优化

不需要奖励模型，直接从偏好数据学习。你的 LLM 本身就是奖励模型。

 方法二

PPO — 复刻 InstructGPT

三步走：SFT → 训练奖励模型 → PPO 微调。OpenAI 的经典方案。

 方法三

GRPO — 复刻 DeepSeek-R1

不需要 Critic，用组相对优势。专为推理任务设计的 RL 方法。
对比

三种方法的选择

DPO 最简单但效果有限；PPO 最成熟但复杂；GRPO 最适合推理任务。

第 11 章

DPO — 直接偏好优化

核心思想

DPO 的核心洞察：你的大语言模型实际上就是一个奖励模型。不需要单独训练奖励模型，直接从偏好数据优化 LLM。

机器学习类比 DPO ↔ 对比学习 / 排序学习
DPO 偏好数据 chosen 和 rejected 成对比较
↔
对比学习正负样本对 anchor + positive vs negative
DPO 拉开 chosen/rejected 概率 让偏好回答概率高于厌恶回答
↔
对比损失拉开正负样本嵌入 让正样本靠近 anchor，负样本远离
类比要点：DPO 和对比学习（如 CLIP、SimCLR）的思路惊人地相似——都是通过"成对比较"学习，拉开好的、推开差的。DPO 拉开的是回答的概率，对比学习拉开的是嵌入空间的距离。这种"相对比较"比"绝对打分"更容易收集数据，因为人类判断"A 比 B 好"比给 A 打 8 分更可靠。

偏好数据集

# 偏好数据格式
{
  "prompt": "这部电影怎么样？",
  "chosen": "这部电影很好看。",      # 人类偏好的回答
  "rejected": "这部电影不好看。"     # 人类厌恶的回答
}

数据集来源：调整温度让 LLM 生成不同回答 → 人工标注偏好；ChatGPT 用户选择；合成数据等。

DPO 数据集的常见问题

正负例区分不明显：chosen 和 rejected 太相似，人类都分不清。
偏好循环：A>B 和 B>A 同时存在，模型无法学到任何东西。

DPO 目标函数

J_DPO(θ; π_ref) = E[ log σ( β · ( log πθ(yw|x)/πθ(yl|x) - log πref(yw|x)/πref(yl|x) ) ) ]

其中 yw 是偏好回答，yl 是厌恶回答，πref 是冻结的参考模型，β 是超参数。完整推导（含 log 商 = log 差的变换、训练动态分析）见DPO目标函数推导。

第 12 章

PPO 微调 LLM — 复刻 InstructGPT

InstructGPT 三步流程

# InstructGPT 训练流程
Step 1: SFT — 有监督微调
  输入: Prompt → 输出: 人工编写的回复

Step 2: RM — 训练奖励模型
  SFT 生成 4-9 个回复 → 人工排序 → 训练 RM

Step 3: RL — PPO 强化学习
  LLM 生成回复 → RM 打分 → PPO 更新参数
  + KL散度惩罚（防止偏离SFT太远）
机器学习类比 InstructGPT 三步 = 监督预训练 + 训练评估器 + 对抗微调
Step 1: SFT 监督微调 学习基本的回答格式
↔
预训练 / 初始化 给模型一个合理的起点
Step 2: RM 训练奖励模型 学习"什么是好回答"
↔
训练评估指标 / 判别器 定义"什么是好模型"
Step 3: PPO 强化学习 优化 LLM 使 RM 打高分
↔
对抗训练 / GAN 优化生成器骗过判别器
类比要点：Step 2+3 的组合本质上就是 GAN 的思路——训练一个判别器（RM）区分好坏，再训练生成器（LLM）骗过判别器。区别在于 GAN 的判别器和生成器同时训练、直接对抗；InstructGPT 是先训好 RM 再用它指导 LLM，且加了 KL 约束防止 LLM 跑偏。这也解释了为什么 DPO 更简单——它跳过了 Step 2，直接用偏好数据做 Step 3。

即时奖励 Rₜ 的计算

在倒立摆环境中，每步奖励简单为 +1。但在 LLM 中，奖励计算更复杂——奖励模型只能给完整序列打分，如何转化为 token 级奖励？

Rₜ = −β · log[ πθ(yₜ|x, y<ₜ) / πref(yₜ|x, y<ₜ) ] (每个 token)

R_T = RM得分 − β · log[ πθ(yₜ|x, y<ₜ) / πref(yₜ|x, y<ₜ) ] (最后一个 token)

分两种情况的精妙设计、KL 惩罚逐 token 计算的原因，以及稀疏奖励如何用 GAE 抹匀，详见即时奖励Rt计算。

关键秘诀：KL 散度惩罚

InstructGPT 成功的关键是 KL 散度惩罚项。它防止奖励黑客（Reward Hacking）：如果没有约束，RL 模型会利用奖励模型的漏洞获得高分但生成无意义文本。KL 惩罚让微调后的模型不偏离原始 SFT 模型太远。

PPO 微调实战

实战包含完整的 InstructGPT 复刻流程：

  • SFT：对 GPT-2 进行指令微调
  • 训练奖励模型：基于排序数据训练
  • ActorCritic 模型：LLM 作为 Actor + 价值网络作为 Critic
  • GAE 计算：广义优势估计
  • PPO 训练循环：采样 → 计算奖励 → 计算优势 → 更新
  • 评估：对比微调前后模型质量

第 13 章

GRPO 微调 LLM — 复刻 DeepSeek-R1

DeepSeek-R1 的突破

DeepSeek 在 DeepSeek-Math 和 DeepSeek-R1 上的突破，展现了 RL 在提升 LLM 推理能力方面的巨大潜力。这得益于 GRPO 算法。

GRPO 目标函数

J^GRPO(θ) = (1/G) Σᵢ (1/|τᵢ|) Σₜ min[ρ·A, clip(ρ, 1-ε, 1+ε)·A] − β·D_KL[πθ‖πref]

使用 DAPO 微调 Qwen2.5-3B

本章使用 DAPO（GRPO 的变种，由字节提出）微调 Qwen2.5-3B-Instruct，从零复刻类似 DeepSeek-R1 的推理模型，产生思维链。

机器学习类比 GRPO 训练推理 = 自监督 + 多路采样投票
对同一问题采样 G 条回答 生成多个候选，比较优劣
↔
多路采样 / Beam Search 生成多个候选，选最优的
用正确性作为奖励 答案对=正奖励，错=负奖励
↔
自监督学习 用数据本身验证模型输出
类比要点：GRPO 训练推理模型的过程像"自监督的多路投票"——对每道题生成多个解答，对的强化、错的削弱，模型逐渐学会"怎么思考才能答对"。这比人类标注思维链成本低得多，这也是 DeepSeek-R1 能低成本获得强推理能力的关键。注意：这和 GAN 的对抗不同，GRPO 没有"判别器"，奖励来自客观的正确性验证（数学题能验证答案）。

DAPO 的两项改进

  • Token 级策略梯度损失：每个 token 的优势等于其所在轨迹的组相对优势
  • 移除 KL 散度：不再需要参考策略网络，减少 GPU 内存

DAPO 算法流程

# DAPO 伪代码
for each training step:
    1. 随机选取 N 个问题: q₁, q₂, ..., qₙ
    2. 对每个问题 qᵢ，采样 G 条回答: aᵢ,₁, ..., aᵢ,G
    3. 计算每条回答的奖励 rᵢ,ⱼ
    4. 组内归一化: μᵢ = mean(rᵢ,₁,...,rᵢ,G)
                 σᵢ = std(rᵢ,₁,...,rᵢ,G)
    5. Token 级优势: Aᵢ,ⱼ[t] = (rᵢ,ⱼ - μᵢ) / σᵢ
    6. 使用 GRPO 目标函数计算梯度并更新

奖励函数设计

推理任务的奖励函数设计至关重要。常见策略：正确性奖励（答案对不对）、格式奖励（是否有思维链格式）、过程奖励（中间步骤质量）。

GRPO 应用场景

场景一

医疗思维链

使用 GRPO 训练 LLM 产生结构化的医疗推理过程，提升诊断准确性。

场景二

Text-To-SQL

用 GRPO 优化 LLM 的 SQL 生成能力，通过执行正确性作为奖励信号。

为什么 GRPO 适合推理任务？

推理任务（数学、代码、逻辑）有明确的正确性标准，适合作为奖励信号。GRPO 的组相对优势天然适合这类"多条采样取最优"的场景——对同一问题生成多个回答，好的强化、差的削弱。
