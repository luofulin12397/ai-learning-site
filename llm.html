第 1 章

microgpt — 200 行代码理解 GPT

"此文件包含了完整算法，其余一切不过是效率优化。" —— Andrej Karpathy

这 200 行代码中运行的数学运算，与 ChatGPT、Claude、Gemini 乃至所有基于 Transformer 架构的语言模型所执行的运算完全一致。区别仅在于规模与速度——而非算法本身。

整体流程

# microgpt 的五步流程
1. DATASET:    Load 32,000 human names ("emma", ...)
2. TOKENIZER: Map each character → integer ID
3. MODEL:      Build a tiny GPT (4,192 parameters)
4. TRAIN:      Show it names, adjust parameters
5. GENERATE:   Ask it to invent new names

训练完成后，模型能生成从未见过的、听起来合理的名字，比如 "Aalina" 或 "Relyn"。它纯粹通过示例学会了英文名字的统计规律。

数据集：文档流

大语言模型的燃料是文本数据流。在生产级应用中，每个文档可能是一个互联网网页，但对于 microgpt，我们使用更简单的示例：32,000 个名字，每行一个。

# 加载数据集
docs = [l.strip() for l in open('input.txt').read().strip().split('\n') if l.strip()]
random.shuffle(docs)
print(f"num docs: {len(docs)}")  # ~32,000

分词器：字符到整数

分词器将文本拆分为离散的 token。在 microgpt 中，每个字符就是一个 token。我们构建一个字符到整数的映射表。

自动微分（Autograd）

PyTorch 的 autograd 机制自动计算梯度。当我们对损失调用 loss.backward() 时，PyTorch 会沿着计算图反向传播，自动计算每个参数的梯度。

核心概念

Autograd 的本质是链式法则的自动执行。你只需要定义前向计算，PyTorch 会自动构建计算图并计算反向梯度。

训练循环

训练过程就是反复执行：前向传播计算损失 → 反向传播计算梯度 → 更新参数。

for step in range(max_steps):
    # 1. 采样一个 batch
    xb, yb = get_batch('train')

    # 2. 前向传播
    logits, loss = model(xb, yb)

    # 3. 反向传播
    optimizer.zero_grad()
    loss.backward()

    # 4. 更新参数
    optimizer.step()

推理：生成新名字

模型训练好后，从一个起始 token 开始，反复预测下一个 token，直到生成结束符。

为什么从 microgpt 开始？

microgpt 用最少的代码揭示了 GPT 的本质：数据集 → 分词器 → 模型 → 训练 → 生成。理解了这个流程，后续的 GPT-2、甚至 ChatGPT 的架构都是在这个基础上的扩展。

第 2 章

LLM 简介 — 核心原理

核心问题：预测下一个 Token

大语言模型要解决的问题是预测下一个 token（next token prediction, NTP）。给定一段文本的前缀，模型预测下一个 token 是什么。详见预测下一个Token。

训练数据：输入-目标对

文本数据 "君不见黄河之水天上来" 被切分为输入-目标对：

输入          预测目标
君不见黄河    → 之
君不见黄河之  → 水
君不见黄河之水 → 天
君不见黄河之水天 → 上

预测目标不是人类标注而来，而是程序通过切分文本自动标注的，所以叫做自监督学习。

模型结构：Decoder-Only Transformer

LLM 采用仅解码器的 Transformer 架构（Decoder-Only Transformer）。核心组件包括：

  • Token 嵌入层 + 位置嵌入层
  • 多头自注意力（Multi-head Self-Attention）
  • 前馈层（Feed Forward）
  • 层归一化（LayerNorm）
  • 残差连接（Residual Connection）

损失函数：交叉熵

LLM 本质上是一个分类模型。从极大似然估计的角度看：

最大化 P(d|abc) · P(c|ab) · P(b|a)
等价于最小化 −log P(d|abc) − log P(c|ab) − log P(b|a)

这就是交叉熵损失函数

给定输入 "ab"，我们希望预测的下一个 token 属于分类 "c" 的概率越大越好。详细推导见交叉熵损失推导。

重要区分

LLM 的"分类"不是传统意义上的图片分类。它的类别是词表中的所有 token，可能高达数万个。模型输出的是对所有 token 的概率分布。

第 3 章

GPT-2 — 完整实现

从 microgpt 到 GPT-2，核心架构不变，但每个组件都更加完整和规范。本章从零实现 GPT-2 并加载 OpenAI 开源的预训练权重。

数据集准备

使用 tiktoken 分词器（GPT-2 的 BPE 分词），通过滑动窗口将文本切割成重叠的序列：

class GPTDatasetV1(Dataset):
    def __init__(self, txt, tokenizer, max_length, stride):
        self.input_ids = []
        self.target_ids = []
        token_ids = tokenizer.encode(txt)

        for i in range(0, len(token_ids) - max_length, stride):
            input_chunk = token_ids[i : i + max_length]
            target_chunk = token_ids[i + 1 : i + max_length + 1]
            self.input_ids.append(torch.tensor(input_chunk))
            self.target_ids.append(torch.tensor(target_chunk))

模型结构定义

GPT-2 模型自顶向下包含：

class GPTModel(nn.Module):
    def __init__(self, cfg):
        super().__init__()
        self.tok_emb = nn.Embedding(cfg["vocab_size"], cfg["emb_dim"])
        self.pos_emb = nn.Embedding(cfg["context_length"], cfg["emb_dim"])
        self.drop_emb = nn.Dropout(cfg["drop_rate"])
        self.trf_blocks = nn.Sequential(
            *[TransformerBlock(cfg) for _ in range(cfg["n_layers"])]
        )
        self.final_norm = LayerNorm(cfg["emb_dim"])
        self.out_head = nn.Linear(cfg["emb_dim"], cfg["vocab_size"], bias=False)

核心组件详解

词嵌入 + 位置编码

Token 嵌入将离散的 token ID 映射为连续向量。位置编码告诉模型每个 token 在序列中的位置。GPT-2 使用可学习的位置嵌入。

多头自注意力

自注意力机制让每个 token "看到" 序列中所有之前的 token。多头注意力将嵌入空间分成多个子空间，并行计算注意力。

Attention(Q, K, V) = softmax(QKᵀ / √dₖ) · V

公式详解和 Q/K/V 的来源见自注意力机制。

前馈层

两层线性变换 + GELU 激活函数，对每个位置独立处理。

层归一化 + 残差连接

每个子层（注意力和前馈）都包裹在 LayerNorm + 残差连接中，稳定深层网络训练。
