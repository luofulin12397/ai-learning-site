第 1 章

microGPT — 60 行代码看懂大模型全貌

"此文件包含了完整算法，其余一切不过是效率优化。" —— Andrej Karpathy

🎯 导读

ChatGPT、Claude、Gemini 这些大模型，核心算法其实只有薄薄一层。这章我们用一个只有 4192 个参数、能学着编人名的迷你模型 microGPT，把"大模型到底在干嘛"一次看穿。看完你会发现：剩下的一切，不过是"把它做大做快"。

🌰 先讲个直觉：接龙游戏

想象你在玩一个接龙游戏：我给你看几万个英文名字（emma、olivia、ava……），不告诉你任何规则。看多了之后，我让你自己编一个新名字。

你大概率会编出 "Aalina" 或 "Relyn"——念起来像名字，但现实中查无此人。你并没有背规则，只是凭语感：哪个字母后面"顺嘴"接哪个字母。

大模型干的就是这件事，只不过：它接龙的不是"字母"，而是"词"（token）；它看的不是几万个名字，而是整个互联网；它的"语感"不是凭感觉，而是几千亿个可调参数。ChatGPT 跟你聊天，本质就是把你俩的对话当成一份没写完的文档，一个词一个词地猜着补全。

整体流程：一条五步流水线

# microGPT 的五步流程
1. DATASET:    加载 32,000 个人名（"emma", "olivia", ...）
2. TOKENIZER: 每个字符 → 整数 ID（翻译官）
3. MODEL:      构建微型 GPT（4,192 个参数）
4. TRAIN:      给它看名字，猜错 → 调参数
5. GENERATE:   让它编新名字

数据集与分词器

microGPT 用最简单的"字符级"分词：26 个小写字母各给一个编号（a→0, b→1……z→25），再加特殊符 BOS。名字 "emma" 就变成数字串 [BOS, 4, 12, 12, 0, BOS]。

# 加载数据集
docs = [l.strip() for l in open('input.txt').read().strip().split('\n') if l.strip()]
random.shuffle(docs)
print(f"num docs: {len(docs)}")  # ~32,000
真正的 GPT-4 用什么分词？

真正的 GPT-4 用 BPE（子词分词），词表约 20 万。常见词如 "the" 是一个 token，生僻词才拆开。原理完全一样：把文字变成整数序列。

自动微分：模型怎么知道往哪改？

训练 = 不停地微调参数，让"错误"变小。问题是：有 4192 个参数，每个该调大还是调小、调多少？

关键问题："如果我把这个参数动一点点，错误会变大还是变小？" 这个"动一点点会怎样"，数学上叫导数 / 梯度。

反向传播的魔法：把每一步运算记成一块"乐高积木"，每块积木知道"输入动一点，输出会动多少"。从最后的"错误"出发，沿着计算路径一路往回乘（链式法则），跑一遍就能算出全部 4192 个梯度。

三个反复出现的小积木

  • softmax：把一串分数变成"加起来等于 1 的概率"
  • rmsnorm：给向量做"标准化体检"，防止数值越滚越大
  • Adam：比普通梯度下降更聪明的优化器——会记住梯度"惯性"，自动给每个参数调步长

训练循环

for step in range(max_steps):
    xb, yb = get_batch('train')          # 1. 取一批名字
    logits, loss = model(xb, yb)         # 2. 猜下一个，算错多少
    optimizer.zero_grad()
    loss.backward()                      # 3. 反向传播，算梯度
    optimizer.step()                     # 4. 朝"错误更小"微调参数

第 2 章

LLM 简介 — 大模型在干嘛：预测下一个词

🎯 导读

上一章用 microGPT 看了全貌，这章把"预测下一个词"讲得更透：大模型到底在解决什么任务？喂给它的训练数据长啥样、为什么不用人标注？它怎么衡量自己猜得好不好？看完你能用一句话向任何人解释"GPT 是怎么工作的"。

🌰 直觉：念诗接龙

念一句诗："君不见黄河之水天上来"。

我念到"君不见黄河"，停住，问你下一个字——你几乎不假思索就说"之"。再念"君不见黄河之"，你说"水"。再念"君不见黄河之水"，你说"天"……

注意：每次你都是只盯着"已经出现的字"，去猜"下一个最该出现的字"。你没有想好整句话，而是一个字一个字往后蹦，每一步都基于前面所有字做判断。大语言模型干的就是这件事，一模一样。

"大模型不是在理解问题然后回答，它是在把你的话当成一篇没写完的文档，一个词一个词地往下补。" —— 理解大模型的终极直觉

你问它"中国的首都是哪？"，它之所以回答"北京"，不是因为它"查到了答案"，而是因为在它念过的海量文字里，"中国的首都是"后面接"北京"的概率最高。就这么回事。

① 核心任务：预测下一个 Token（NTP）

先说 token：模型不是按"字"或"词"处理文字，而是按一种切碎的小块。中文里它可能差不多是一个字，英文里 "playing" 可能被切成 "play"+"ing"。Token 就是模型眼里的一个基本单位。

给它  "君不见黄河"      →  它猜  "之"
给它  "君不见黄河之"     →  它猜  "水"
给它  "君不见黄河之水"    →  它猜  "天"
# 每猜出一个就接到末尾，作为新输入再猜下一个。循环往复。

② 自监督学习：数据自己当老师

训练 AI 最烦的就是标注。但"预测下一个词"有个天大的便宜：答案藏在文本自己身上。

拿"君不见黄河之水天上来"这句话，不用人来标注，程序自动切：

输入（模型看到的）          预测目标（正确答案）
君不见黄河                之
君不见黄河之              水
君不见黄河之水            天
君不见黄河之水天          上
# 每个"正确答案"就是原文的下一个字，根本不用人标！

整个互联网的文本，都能这样自动变成无穷无尽的练习题。这种"标准答案是数据自己给的"的玩法，就叫自监督学习——它是大模型能"吃下整个互联网"的关键前提。

③ 滑动窗口：喂饱 GPU

实际训练不会真从"君"一个字开始加长。为了喂饱 GPU 的并行能力，数据用一个固定长度的框去截取：

窗口长度 = 5：
[君 不 见 黄 河]  →  目标是 [不 见 黄 河 之]
[不 见 黄 河 之]  →  目标是 [见 黄 河 之 水]
# 一个窗口产生 5 道练习题（每个位置都在预测下一个），GPU 并行算！

这个框的长度就是上下文长度（Context Length）——模型一次最多能"往回看"几个 token。GPT-2 是 1024，今天是几十万。框越大，模型能记住的前文越长。

④ 模型结构：Decoder-Only Transformer

名字很唬人，但你已经见过它的全部零件了——嵌入层、一摞"注意力 + MLP"积木、输出层。它的设计天生为"从左往右、看着前文猜下一个"服务。一条铁律：只能看前面、不许偷看后面（因为要预测的就是后面那个词）。

下一章会把 GPT-2 这个结构彻底拆开。

📐 损失函数：交叉熵

预测下一个 token，本质上是一道"超大号选择题"。模型每次从整个词表（几万个候选）里选一个。所以输出不是"一个词"，而是给词表里每个候选词打一个概率分。

假设训练数据是 "abcd"，我们希望模型：看到 "abc" 时给 "d" 的概率高，看到 "ab" 时给 "c" 的概率高……把这些概率乘起来，越大越好：

①
P(d|abc) · P(c|ab) · P(b|a) → 最大化
②
取 log（单调递增，不改变最优解在哪）：log P(d|abc) + log P(c|ab) + log P(b|a)
③
加负号变最小化：−log P(d|abc) − log P(c|ab) − log P(b|a) = 交叉熵损失

人话版：模型给正确答案的概率越高（越不惊讶），损失越小。给 100% 把握 → 0 损失；几乎没料到 → 损失爆炸。训练就是让这个"惊讶程度"越来越小。

那"正确答案"长什么样？独热编码

假设词表只有 a、b、c，正确答案是 c → 独热编码 = [0, 0, 1]。模型预测可能是 [0.8, 0.1, 0.1]——它以为多半是 a。给正确答案只打了 0.1 的概率 → 损失 = −log(0.1) ≈ 2.3，很大，该挨训了。

详细推导见交叉熵损失推导。

💻 代码主干

# ① 自监督：自动切"输入→目标"练习题
def make_pairs(tokens, window):
    pairs = []
    for i in range(len(tokens) - window):
        x = tokens[i : i + window]          # 输入：连续 window 个 token
        y = tokens[i + 1 : i + window + 1]  # 目标：整体往右挪一格
        pairs.append((x, y))                # 注意：没人标注！
    return pairs

# ② 算一句话的交叉熵损失
def loss_of_sequence(model, x, y):
    total = 0
    for i in range(len(x)):
        logits = model(x[: i + 1])           # 看着前 i+1 个 token
        probs  = softmax(logits)             # 变成概率分布
        total += -log(probs[y[i]])           # 正确答案概率越低，罚越多
    return total / len(x)
🧠 产品经理视角

「一切皆预测下一个词」是理解大模型的总钥匙。翻译、写代码、做总结——内部全是同一个动作。所谓"让模型做某件事"，本质是设计好 prompt，把你想要的轨道铺好。

它没有"对错"概念，只有"像不像"。模型优化的是"接得像不像训练数据"，不是"对不对"。所以它能流畅编造不存在的事——幻觉的根源就在损失函数里。

上下文长度是硬约束。模型一次只能看进窗口那么多 token，超出的旧内容真看不见。长文档问答必须考虑切片、检索、摘要。

自监督是大模型便宜又强大的根本原因。训练答案是文本自己提供的，所以能无限扩大数据规模——"喂更多数据几乎总能让模型更强"（Scaling Law 的前提）。

📌 一句话记住：大模型只会做一件事——把你的话当成没写完的文档，从词表里挑概率最高的下一个 token 接上去；训练就是用"右移一格"自动造海量练习题，再用交叉熵损失逼它"少惊讶、多猜对"。

第 3 章

GPT-2 — 把 Transformer 拆开看

🎯 导读

microGPT 是"60 行看全貌"的玩具，GPT-2 是"把每个零件焊到工业级"的工厂。这一章从零实现 GPT-2（1.24 亿参数起步）并加载 OpenAI 开源的预训练权重——把 6 个核心零件逐个拆开看清楚。看完你会明白：GPT-2 和今天的 GPT-4，零件清单几乎一模一样，只是堆得更多更大。

🌰 直觉：一条信息加工流水线

假想你是一条流水线的厂长，要把"一句话"变成"下一个词的预测"：

🏭 ① 翻译工位：把文字（token）翻译成数字向量——因为机器只认数字。这叫词嵌入。

🏷️ ② 贴座位号工位：给每个词贴一个"第几位"标签——"狗咬人"和"人咬狗"完全不同。这叫位置编码。

⚙️ ③ 核心车间（重复 N 遍）：每个词先抬头看前面的词（注意力），把相关信息搬过来；再低头自己琢磨（前馈层）。一"看"一"想"组成一个Transformer 块，GPT-2 叠了 12 层。

📦 ④ 出货工位：给词表里 5 万多个词每个打一个分，分最高的就是"下一个词"。

① 词嵌入 — "给每个词配一个坐标"

人话定义：一张超大的查找表。词表 50257 个词，每个对应一行 768 个浮点数。给我一个 token ID，我就返回那行的 768 个数——这串数就是这个词的含义向量。

训练后，意思相近的词坐标会靠得近（"国王"和"女王"离得近）。

嵌入层 = 一种省力的矩阵乘法

nn.Embedding 做的事，完全等价于"把 token 变成独热向量再乘权重矩阵"。但独热向量里 99.99% 都是 0，乘起来全是"乘 0 的无效计算"。所以工程上直接用查表代替矩阵乘——结果一样，快得多。

② 位置编码 — "座位号"

人话定义：又一张查找表，按"位置"查。第 0 个位置一个 768 维向量，第 1 个一个……一直到上下文长度。

为什么需要它：注意力机制不分先后——它眼里所有词是一袋子混在一起，没有顺序概念。但语言里顺序是命根子。所以我们给每个位置一个向量，直接加到词嵌入上，相当于在每个词的坐标里掺进"我排第几"的信息。GPT-2 的位置编码是训练出来的，不是人工公式。

"我是左元" vs "我是莎士比亚" — 说明为什么还不够

这两个句子里第一个"我"，token 相同、位置相同，所以词嵌入+位置编码一模一样！但这两个"我"指的人不同。怎么区分？靠后面的注意力——让"我"去看后面的"左元/莎士比亚"，从而获得不同含义。这就引出了注意力机制的必要性。

③ 仅解码器 Transformer — "叠 12 层积木"

GPT-2 把同一种"积木块"（TransformerBlock）叠了 12 层。一层只能做有限的"看+想"，叠 12 层信息能反复"看了再想、想完再看"，逐层抽象出越来越高级的语义。

一条铁律：每个块的输入和输出形状必须完全一样——都是 (批量, 序列长度, 768)。否则上一块的输出就接不进下一块。

每个块内部是固定模式：先归一化→子层→加回残差。残差连接 = 留一条"高速直达通道"，叠几十层也不会把原始信息弄丢。

④ 层归一化 — "给每个词做标准化体检"

人话定义：把一个词的 768 个数字，减掉平均值、再除以波动幅度，拉成"均值 0、波动 1"的标准状态；再用两个可学习参数 γ、β 缩放平移回合适的尺度。

为什么需要：深层网络里数字传着传着容易"越滚越大"或"越缩越小"。归一化就像每过一关给数据"校准一次仪表盘"。microGPT 用 rmsnorm，GPT-2 用 LayerNorm，思路一样：稳住数值、稳住训练。

y = (x − E[x]) / √(Var[x] + ε) · γ + β

⑤ 多头注意力 — "开 12 个分身同时找重点"

这是整个 Transformer 的灵魂。先记住直觉：注意力是 token 之间唯一能互相通信的地方。

每个词带着三样东西：Query（我在找什么）、Key（我有什么）、Value（被选中我能给什么）。当前词拿自己的 Q 去和前面所有词的 K 对暗号，谁对得上就多看谁一眼，把它的 V 信息拿过来。

Attention(Q, K, V) = softmax(QKᵀ / √dₖ) · V

"多头"是不止一个注意力，而是拆成 12 个"分身（头）"并行工作，每个头关注不同类型的关系（有的盯语法、有的盯指代……），最后把各头的发现拼回来。

因果掩码 — "不能偷看后面的答案"

算分时把"未来的词"那部分分数填成 −∞，softmax 后权重就是 0。这样每个词只能看到自己和前面的词。考试不能偷看后面的答案——掩码就是蒙住"未来"的那块布。

⑥ 前馈层 FFN — "看完之后自己消化一下"

人话定义：一个小小的两层全连接网络，对每个词单独做一次"放大→筛选→压缩"的非线性加工。

注意力只会做"加权平均"——把别人信息按权重搬过来混在一起，本质是线性操作。FFN负责把混来的信息加工、判断、抽象。

"开会时，注意力是听取大家意见，FFN 是自己回去消化一下。不然只听不想，会议纪要很热闹，决策不一定靠谱。"
FFN(x) = W₂ · σ(W₁x + b₁) + b₂
W₁：768→3072 放大；σ(GELU)：非线性筛选；W₂：3072→768 压回
📐 为什么 FFN 的参数比注意力还多？（可展开）

设模型维度 d、FFN 中间维度 d_ff = 4d：

FFN 两个矩阵：d×4d + 4d×d = 8d²

注意力 Q、K、V、O 四个矩阵：4d²

所以单论参数，FFN 是注意力的约 2 倍。业界普遍认为模型大量的"知识"和"事实记忆"存在 FFN 里——注意力负责"从哪取信息"，FFN 负责"把信息变成什么"。

class GPTModel(nn.Module):
    def __init__(self, cfg):
        super().__init__()
        self.tok_emb = nn.Embedding(cfg["vocab_size"], cfg["emb_dim"])
        self.pos_emb = nn.Embedding(cfg["context_length"], cfg["emb_dim"])

一个 TransformerBlock 的固定节奏：

def transformer_block(x):
    x = x + attention(layernorm(x))   # 子层1：看别人（残差直通）
    x = x + ffn(layernorm(x))         # 子层2：自己想（残差直通）
    return x                          # 形状和输入完全一致

多头注意力核心（最容易绕晕的"切头/拼头"）：

def multi_head_attention(x):     # x: (批量b, 词数t, 768)
    Q, K, V = W_q(x), W_k(x), W_v(x) # 线形变换出 Q/K/V

    # 把 768 切成 12 个头，每头 64 维：(b,t,768) → (b,12,t,64)
    Q, K, V = split_heads(Q), split_heads(K), split_heads(V)

    scores  = Q @ K.transpose()      # 每头各算一遍：词与词的相关分
    scores  = mask_future(scores)     # 因果掩码：未来填 -inf
    weights = softmax(scores / 64**0.5) # 缩放 + 归一成权重

    out = weights @ V                 # 按权重搬运信息
    out = merge_heads(out)            # 12 个头拼回 768 维
    return W_o(out)                  # 最后再做一次线性投影

🧠 产品经理视角

"零件清单"是通用的。词嵌入 + 位置编码 + N层(注意力+FFN+归一化+残差) + 输出头——从 GPT-2 到 GPT-4 基本没换。后来的改进（RoPE、GQA、SwiGLU）都是在固定零件上换型号。理解了 GPT-2，你就有了看懂所有新模型架构的"底图"。

优先用开源权重，别从头训练。架构代码一样时，权重可以直接灌进去。99% 的情况优先用现成的开源/闭源基座模型，把钱花在微调和应用层。

上下文长度是硬上限。GPT-2 是 1024 token，今天几十万——但注意力要看的词越多，算力消耗涨得越快。长上下文一直是成本痛点。

FFN 存知识，注意力查信息。理解这个分工有助于理解模型的行为——比如为什么模型有时"知道"一个事实但"注意不到"它。

📌 一句话记住：GPT-2 = microGPT 的工业放大版：查表→叠 N 层（看别人 + 自己想）→打分猜下一个词。零件清单从 GPT-2 到 GPT-4 基本没变，只是越堆越大。而你大概率永远不用从头训练——下载开源权重直接用就行。
