强化学习

从智能体与环境交互的基本概念出发，逐步推导策略梯度法、REINFORCE、Actor-Critic、PPO 直到 GRPO。这是理解 RLHF 和 DeepSeek-R1 的理论基石。

第 4 章

强化学习基础

核心概念：智能体与环境

强化学习讨论的问题是智能体（agent）怎么在复杂、不确定的环境中最大化它能获得的奖励。

核心循环：智能体获取环境状态 → 根据状态采取动作 → 环境转移到新状态 → 反馈即时奖励。

关键术语

Agent（智能体）· Environment（环境）· State（状态）· Action（动作）· Reward（即时奖励）· Policy（策略）

倒立摆环境

本书用倒立摆（CartPole）作为贯穿始终的示例环境。智能体是推车，目标是通过左右移动保持木杆不倒。

  • 动作空间：向左推 / 向右推
  • 状态：推车位置、速度、木杆角度、角速度
  • 即时奖励：木杆不倒 = +1
  • 结束条件：杆倒下 / 推出屏幕 / 坚持 200 步（成功）

策略函数

策略是智能体的动作模型，决定了在给定状态下采取什么动作。策略是一个函数：

π(a|s) = p(aₜ = a | sₜ = s)

输入状态 s，输出动作的概率分布

策略不一定是确定性的——往往是从概率分布中采样动作。这涉及探索与利用的平衡。

机器学习类比 用监督学习理解 RL 基本概念
状态 S / 动作 A 环境描述 / 智能体的选择
↔
特征 X / 标签 ỹ 数据描述 / 模型的预测
策略函数 π(a|s) 输入状态→输出动作概率分布
↔
分类器 / softmax 输出 输入特征→输出类别概率分布
类比要点：策略网络和分类器的网络结构几乎一样——都是全连接层 + softmax。区别在于：分类器的"正确答案"来自人工标注（ground truth），而策略的"正确答案"由回报信号反推。状态就是 RL 中的"输入特征"，动作就是 RL 中的"预测输出"。
探索 vs 利用

如果每次都选概率最大的动作（利用），可能陷入局部最优。偶尔选择概率低的动作（探索），可能发现更好的长期策略。就像去熟悉的餐厅（安全）还是尝试新餐厅（可能惊喜也可能失望）。

价值函数

价值函数衡量在某个状态下，未来能获得的期望累积奖励。状态价值函数 V(s) 和动作价值函数 Q(s,a) 是 RL 的核心概念。

机器学习类比 价值函数 = 模型期望表现
价值函数 V(s) 从状态 s 出发的期望回报
↔
模型期望准确率 在特征 x 上的期望表现
优势 A = Q − V 动作相对基准的好坏
↔
残差 = 实际 − 预测 预测偏离基准的程度
类比要点：价值函数告诉你"当前状态本身值多少分"（基准线），动作的好坏要相对这个基准来衡量。就像在监督学习中，残差衡量"实际值偏离预测多少"——优势函数衡量"这个动作比平均水平好多少"。

马尔可夫决策过程（MDP）

MDP 是 RL 的数学框架，包含：状态集 S、动作集 A、转移概率 P、奖励函数 R、折扣因子 γ。

MDP 的目标是找到一个策略 π，使得期望累积奖励最大化。

贝尔曼方程

贝尔曼方程是 RL 的核心递推关系，将价值函数分解为即时奖励 + 下一状态的价值：

V(s) = E[Rₜ + γ·V(Sₜ₊₁) | Sₜ = s]

推导过程和动作价值函数 Q(s,a) 的贝尔曼方程见贝尔曼方程详解。

机器学习类比 贝尔曼方程 = 递归的损失分解
贝尔曼方程 V(s) = E[Rₜ + γV(Sₜ₊₁)]
↔
残差连接 / 梯度反传 当前损失 = 即时误差 + 未来误差的反传
类比要点：贝尔曼方程把"长期累积回报"分解为"即时奖励 + 下一状态价值"，就像残差网络把深层误差分解为"当前层误差 + 传给前一层的误差"。这种递归结构让 RL 能像反向传播一样逐步更新价值估计。
第 5 章

策略梯度法

核心思想

策略梯度法的核心：将策略函数建模为神经网络，用梯度上升直接优化策略参数 θ，使期望回报最大化。

∇θ J(θ) = Eτ~πθ [ G(τ) · ∇θ log πθ(Aₜ|Sₜ) ]

完整的策略梯度定理推导（含 log-导数技巧）见策略梯度定理。

机器学习类比 策略梯度 ↔ 梯度下降，奖励 ↔ 负损失
奖励 R（越大越好） 环境的即时反馈
↔
−损失 L（越大越好） 预测误差的负值
策略梯度 ∇log π(a|s) 增大好动作的概率
↔
梯度下降 ∇θ L 减小预测误差
类比要点：监督学习有 ground truth → 算交叉熵 → 反向传播 → 梯度下降减小损失。RL 没有 ground truth → 采样轨迹 → 用回报 G 作为权重 → 梯度上升增大好动作概率。两者优化机制本质相同，区别仅在于梯度的来源：监督学习的梯度来自标注，RL 的梯度来自回报信号。记住：最大化奖励 ⟺ 最小化负奖励 ⟺ 最小化损失。

REINFORCE 算法

REINFORCE 是最基础的策略梯度算法：采集一条完整轨迹 → 计算累积回报 → 更新策略。其改进是用 Gₜ 替代 G(τ)，去除时刻 t 之前的奖励噪声。详见REINFORCE推导。

def reinforce(agent, env, num_episodes):
    for episode in range(num_episodes):
        # 1. 采集一条完整轨迹
        trajectory = agent.collect_trajectory(env)

        # 2. 计算每步的累积回报 G(t)
        returns = compute_returns(trajectory)

        # 3. 计算策略梯度并更新
        loss = -sum(log_prob * G for log_prob, G in zip(trajectory.log_probs, returns))
        loss.backward()
        optimizer.step()

带基线的策略梯度

原始策略梯度方差很大。引入基线（baseline）可以减小方差而不改变梯度的期望。常用基线是价值函数 V(s)。

Actor-Critic 架构

Actor-Critic 结合了策略梯度和价值函数方法：

  • Actor（演员）：策略网络 πθ(a|s)，决定采取什么动作
  • Critic（评论家）：价值网络 V(s)，评估当前策略的好坏

Actor 根据 Critic 的反馈调整策略，Critic 根据实际回报调整评估。

机器学习类比 Actor-Critic ↔ GAN 生成器-判别器
Actor（演员） 策略网络，负责"产出"动作
↔
GAN 生成器 负责"造假"生成数据
Critic（评论家） 价值网络，负责"评估"动作好坏
↔
GAN 判别器 负责"鉴别"真假数据
类比要点：两者都是"博弈式协同"——一方生成、一方评估，互相促进。Actor 根据 Critic 的反馈改进策略，Critic 根据实际回报修正评估；正如生成器骗判别器、判别器识破生成器。这个类比在 RLHF 中尤为直观：LLM 是 Actor（生成回答），奖励模型是 Critic（评估好坏），两者对抗式优化。

广义优势估计（GAE）

GAE 通过多步 TD 误差的指数加权平均，在偏差和方差之间取得平衡：

Aₜ^GAE = Σₗ (γλ)ˡ · δₜ₊ₗ

其中 δₜ = Rₜ + γV(Sₜ₊₁) - V(Sₜ) 是 TD 误差

λ 参数的作用和展开形式见GAE详解。

策略梯度法的致命问题

策略梯度法的训练数据是当前策略采样的轨迹。如果一次更新步子太大，新策略可能比旧策略差很多，而且很难恢复——因为差的策略会产生差的数据。这就是 PPO 要解决的问题。

第 6-7 章

PPO — 近端策略优化

策略梯度法的问题

策略梯度法的核心问题：步子迈太大会导致策略崩溃。在深度学习中，训练数据是固定的；但在 RL 中，训练数据是策略采样的——策略变差 → 数据变差 → 策略更差，形成恶性循环。

PPO 的核心思想

PPO 通过置信域方法（Trust Region Method）限制策略更新的幅度，保证新策略不会偏离旧策略太远。

机器学习类比 PPO 的 clip ↔ 学习率调度 / 梯度裁剪
PPO 的 clip(ρ, 1−ε, 1+ε) 限制新旧策略概率比的波动
↔
梯度裁剪 / 学习率衰减 限制单次参数更新的幅度
类比要点：监督学习中用梯度裁剪防止梯度爆炸、用学习率调度控制训练步幅；PPO 用 clip 操作限制策略更新幅度，防止"步子太大扯着蛋"。核心都是限制单次更新的幅度以保证稳定性。区别在于：监督学习的训练数据固定，步子大了顶多这一 batch 训练不好；RL 的训练数据是策略采样的，步子太大策略崩溃 → 数据变差 → 越崩溃越难恢复。

PPO 的替代目标函数

J^PPO(θ) = E[ min( ρ·A, clip(ρ, 1-ε, 1+ε)·A ) ]

其中 ρ = πθ(a|s) / πθ_old(a|s) 是重要性采样比率

clip 操作将重要性采样比率限制在 [1-ε, 1+ε] 范围内，防止单次更新过大。完整推导（含 CPI 目标和 min 操作的作用）见PPO替代目标推导。

PPO 实战

定义 Actor（策略网络）和 Critic（价值网络）：

class PolicyNet(nn.Module):
    def __init__(self, action_size):
        super().__init__()
        self.l1 = nn.Linear(4, 128)
        self.l2 = nn.Linear(128, action_size)

    def forward(self, x):
        x = F.relu(self.l1(x))
        x = F.softmax(self.l2(x), dim=1)
        return x

class ValueNet(nn.Module):
    def __init__(self):
        super().__init__()
        self.l1 = nn.Linear(4, 128)
        self.l2 = nn.Linear(128, 1)

    def forward(self, x):
        x = F.relu(self.l1(x))
        x = self.l2(x)
        return x
PPO 的地位

PPO 是目前最广泛使用的 RL 算法之一。OpenAI 的 InstructGPT 使用 PPO 进行 RLHF，它也是许多游戏 AI 和机器人控制的基础。

第 8 章

PPO 背后的数学

KL 散度

KL 散度衡量两个概率分布的差异：

D_KL(p‖q) = Σ p(x) log [p(x)/q(x)]
  • 两个分布差异越大，KL 散度越大
  • KL 散度 ≥ 0，当且仅当两分布相同时为 0
  • KL 散度是非对称的：D_KL(p‖q) ≠ D_KL(q‖p)

定义、抛硬币实例和三大性质详解见KL散度。

机器学习类比 KL 散度惩罚 ↔ L1/L2 正则化
KL 散度惩罚 β·D_KL[πθ‖πref] 防止策略偏离参考模型太远
↔
L1 / L2 正则化 λ·||w||² 防止权重过大导致过拟合
类比要点：两者都是加在目标函数上的"约束项"，防止模型走极端。正则化防止权重过大 → 过拟合训练集噪声；KL 惩罚防止策略偏离参考模型 → 奖励黑客（Reward Hacking）。在 RLHF 中，没有 KL 惩罚，模型会生成乱码骗高分；就像没有正则化，模型会记住训练集的噪声。超参数 β 和 λ 的作用也对应：太大欠拟合，太小过拟合。

重要性采样

重要性采样允许用旧策略采样的数据来更新新策略，通过重要性权重修正分布偏差：

E_{x~p}[f(x)] = E_{x~q}[ f(x) · p(x)/q(x) ]

方差问题、与 PPO 的关系详见重要性采样。

替代目标

PPO 的理论基础来自单调改进理论：通过限制新旧策略的 KL 散度，保证策略的单调改进。

为什么需要 KL 散度？

在 RLHF 中，KL 散度惩罚项防止微调后的 LLM 偏离原始模型太远。没有它，模型会"奖励黑客"——利用奖励模型的漏洞获得高分但生成无意义文本。

第 9 章

GRPO — 组相对策略优化

GRPO 原理

GRPO 是 DeepSeek 提出的 RL 算法，专为 LLM 推理任务设计。与 PPO 的关键区别：不需要 Critic 网络，而是用一组轨迹的相对优势来估计 baseline。

J^GRPO(θ) = (1/G) Σᵢ (1/|τᵢ|) Σₜ min[ρ·A, clip(ρ, 1-ε, 1+ε)·A]

其中 ρ = πθ(a|s) / πθ_old(a|s)

组相对优势

对每个问题采样 G 条轨迹，计算每条轨迹的奖励，然后用组内归一化计算优势：

Aᵢ = (rᵢ - μ) / σ

μ = mean(r₁, ..., rG), σ = std(r₁, ..., rG)

这种方法简单有效：表现好于平均的轨迹获得正优势，表现差于平均的获得负优势。计算流程与 DAPO 改进见GRPO组相对优势。

机器学习类比 GRPO 组相对优势 ↔ Batch 内归一化
GRPO 优势 A = (r − μ) / σ 组内轨迹奖励的归一化
↔
Batch Normalization batch 内特征的归一化
不用 Critic，用组均值替代 用同伴表现作为基准
↔
不用 running mean，用 batch 均值 用当前 batch 统计量替代
类比要点：GRPO 不训练独立的 Critic 网络估计价值，而是用"同一批采样的多条轨迹的统计量"作为基准——就像 BatchNorm 不维护全局均值，而用当前 batch 的统计量。这种"用同伴做基准"的思路节省了额外网络的训练成本，特别适合参数量巨大的 LLM 微调场景。
GRPO 的优势

不需要训练额外的 Critic 网络，节省 GPU 内存。这对于参数量巨大的 LLM 微调尤为重要。DeepSeek-R1 正是使用 GRPO 实现了推理能力的突破。
