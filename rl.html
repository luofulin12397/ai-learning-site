第 4 章

强化学习是什么

🌰 先讲个直觉：学骑自行车

想象你在学骑自行车。没人能给你一本《骑车说明书》照着背——你只能骑上去，摔，再骑，再摔。摔了你就知道"刚才那样不行"，没摔你就知道"这个感觉对了"。摔了几十次之后，你的身体莫名其妙就学会了，但你说不出具体规则。

这就是强化学习。它和大模型"背书式"学习完全不同：

大模型学习（监督） = 给你看标准答案 → 照着学
强化学习（RL）    = 没人告诉你标准答案 → 做完才打分 → 分高多做、分低少做

再举个更日常的例子：你怎么知道哪家餐馆好吃？没人发给你一张"全城餐馆排行榜"。你只能一家一家去试——吃到好吃的记住、下次还来（这叫利用），偶尔也得尝尝没去过的新店（这叫探索），万一挖到宝呢。强化学习里的 AI，干的就是"骑车 + 找餐馆"这件事：在一个会给它打分的环境里不停地试，慢慢摸索出一套"怎么做能拿最高分"的本事。

核心概念：智能体与环境的互动

RL 的世界只有两个主角，一直在你来我往地互动：

  • 智能体（Agent）：做决策的家伙——我们要训练的 AI。比如自行车上的你、下棋的 AlphaGo。
  • 环境（Environment）：智能体所处的世界——会对动作做出反应。比如自行车+路面、棋盘+对手。

互动循环：智能体看一眼状态 → 做一个动作 → 环境变个样 → 给智能体打个分。然后重复。

关键术语（说人话版）

状态 State：环境此刻的"快照"。倒立摆里是 4 个数字——车位置、车速、杆角度、杆角速度。如果你能看到环境的全部信息，叫"状态"；如果只能看到一部分（打牌只能看自己牌），叫"观察（Observation）"——入门先当一回事。
动作 Action：智能体能做的选择。倒立摆只有两个——向左推、向右推。所有可选动作叫动作空间。
即时奖励 Reward：环境当场给的打分。倒立摆就是"杆没倒 = +1"。
策略 Policy：智能体的"决策大脑"。输入状态，输出"该做什么动作"的概率分布——写成 π(a|s)，读作"在状态 s 下选动作 a 的概率"。为什么用概率？因为要留探索空间。
马尔可夫性质：未来只看现在，不翻旧账。下一个状态只取决于当前状态和动作——这是为了让问题简单到能算。

倒立摆（CartPole）：贯穿始终的示例

一根木杆竖在小推车上（像手心顶扫帚），杆会往一边倒，你得左右推车保持不倒：

  • 动作空间：向左推 / 向右推（2 个离散动作）
  • 状态：推车位置、速度、木杆角度、角速度（4 个数字）
  • 即时奖励：木杆不倒 = +1
  • 结束条件：杆倒 / 推出屏幕（失败）；撑满 200 步（满分）

探索 vs 利用

为什么策略要用"概率抽签"而不是每次都选概率最大的动作？因为只盯着眼前最优，可能错过更好的长远收益。就像前面"找餐馆"的道理：每次都去最熟的那家（利用），你永远发现不了可能更好吃的新店（探索）。倒立摆里也一样：有时候故意推一个"看起来不太对"的方向，反而能让杆子进入一个更容易长期保持平衡的姿势。好的智能体要在"稳吃眼前"和"赌一把未来"之间找平衡。

机器学习类比用监督学习理解 RL 基本概念
状态 S / 动作 A环境描述 / 智能体的选择
↔
特征 X / 标签 ỹ数据描述 / 模型的预测
策略函数 π(a|s)输入状态→输出动作概率分布
↔
分类器 / softmax 输出输入特征→输出类别概率分布
类比要点：策略网络和分类器结构几乎一样——全连接层 + softmax。区别：分类器的"正确答案"来自人工标注，策略的"正确答案"由回报信号反推。把 RL 想象成一个"没有标签的分类问题"会很好理解。

回报与折扣因子

智能体想最大化的不是某一步的奖励，而是从此刻起未来所有奖励的总和——越远的越打折：

Gₜ = Rₜ + γRₜ₊₁ + γ²Rₜ₊₂ + ... = Rₜ + γGₜ₊₁

右边 Gₜ = Rₜ + γGₜ₊₁ 是个漂亮的递推式：今天的总收益 = 今天的奖励 + 打个折的"明天起的总收益"。后面所有 RL 算法都靠它。γ（折扣因子，0~1）有两个作用：让眼前的奖励更值钱（"今天 1 万 vs 一年后 2 万"）；防止无限游戏的总分加到无穷大。

马尔可夫决策过程（MDP）

把"状态转移 + 奖励 + 策略 + 马尔可夫性质"打包在一起，就是 RL 的标准数学框架——MDP。它包含：

  • 状态转移概率 p(s′|s,a)：在状态 s 做动作 a，跳到 s′ 的概率——这是环境说了算，智能体管不着
  • 奖励函数 r(s,a,s′)：跳过去后拿多少分
  • 策略 π(a|s)：智能体唯一能控制的东西

马尔可夫性质：预测下一步只需要看"上一步的状态和动作"——不翻旧账。这是为了让问题简单到能算，否则要考虑全部历史，组合数会爆炸。

贝尔曼方程 — RL 的基石

状态价值函数 V(s)：站在状态 s，按策略 π 一直玩下去，平均能拿多少总分。贝尔曼方程把"现在的价值"和"未来的价值"连了起来：

V(s) = E[ Rₜ + γ · V(Sₜ₊₁) | Sₜ = s ]

人话版：这个状态的价值 = 当场拿到的奖励 + 打个折的"下一个状态的价值"。

换成职场版秒懂：你对一份工作的总评估 = 这个月发的工资 + 打个折的"你对这份工作未来的评估"。它把一个"要算到天荒地老的总和"变成了"只看一步 + 递归"——这正是它伟大的地方。

📐 贝尔曼方程完整推导（可展开）

从 Gₜ = Rₜ + γGₜ₊₁ 出发，代入价值函数定义：

v(s) = E[Gₜ|Sₜ=s] = E[Rₜ|Sₜ=s] + γE[Gₜ₊₁|Sₜ=s]

第一项（即时奖励期望）把所有可能的"动作×转移×奖励"加权求和：

E[Rₜ|Sₜ=s] = Σₐ Σₛ′ π(a|s) p(s′|s,a) r(s,a,s′)

第二项利用马尔可夫性质化成下一个状态的价值：

E[Gₜ₊₁|Sₜ=s] = Σₐ Σₛ′ π(a|s) p(s′|s,a) v(s′)

合起来就是完整的贝尔曼方程。别记推导——记住"现在的价值 = 即时奖励 + 折扣×未来价值"才是地基。

机器学习类比贝尔曼方程 = 递归的损失分解
贝尔曼方程V(s) = E[Rₜ + γV(Sₜ₊₁)]
↔
残差网络 / 梯度反传当前损失 = 即时误差 + 未来误差反传
类比要点：贝尔曼方程把"长期累积回报"分解为"即时+未来"，就像残差网络把深层误差递归分解。

💻 倒立摆代码骨架

import gym, random
env = gym.make("CartPole-v0")
state = env.reset()
done = False; gamma = 0.95
episode_rewards = []

while not done:
    action = random.choice([0, 1])        # 随机智能体：瞎推
    state, reward, done, _ = env.step(action)
    episode_rewards.append(reward)

# 逆序算总收益 Gₜ = Rₜ + γ·Gₜ₊₁
total_return = 0
for r in reversed(episode_rewards):
    total_return = r + gamma * total_return

🧠 产品经理视角

四个关键洞察

1. 判断问题适不适合 RL 就一句话：你能不能给结果打个分，但说不清"标准答案"长什么样？下棋（赢了才知道）、推荐（用户点了才知道）、RLHF（回答让人满意才知道）——全是这个味儿。

2. 奖励函数是 RL 里最难、最坑的部分。两个经典翻车：奖励黑客（AI 找到歪门邪道刷分，比如清洁机器人把垃圾倒出来再捡一遍）；奖励稀疏（太难拿分，AI 试半天学不动）。在 RLHF 里，"奖励"来自人类偏好或奖励模型——这个信号的质量直接决定模型行为上限。

3. ChatGPT 为什么这么会聊天，一半功劳在 RL。预训练让它"会说话"，但"说得讨不讨人喜欢"是靠 RLHF 调出来的——本质就是把"人类偏好"做成奖励信号。

4. RL 是通往 AGI 最有希望的一条路。模仿学习的天花板是"人类已经会的东西"。而 RL 能让 AI 通过自我试错超越人类——AlphaGo 下出"神之一手"，靠的就是它。把策略函数换成神经网络，就是当今 AI 界最性感的方向：深度强化学习。

策略梯度法 — 让策略自己"进化"

🎯 导读

上一章知道了"RL 要训练一个会做决策的智能体"，但到底怎么训？这章给出最主流的路线——策略梯度法。我们会顺着一条改进链走下来：原始策略梯度 → REINFORCE → 带基线 → Actor-Critic → GAE，每一步只解决上一步的一个毛病。看完你能看懂 RLHF、PPO、GRPO 这些词到底在折腾什么。

🌰 直觉：教新手玩顶杆子

想象教新手玩倒立摆。最朴素的办法：让他放手玩一整局，记下总分。

分高 → "你刚才那一连串操作整体不错，以后多这么干"。分低 → "刚才那套以后少这么干"。注意你没有告诉他"第 37 步该往左"——因为你也不知道标准答案。你只是根据最终结果，把他做过的所有动作整体抬或压。玩几千局后，高分动作越来越频繁，他就变强了。

策略梯度法干的就是这件事。"直觉" = 策略网络 πθ，"往上推/往下压" = 用梯度微调参数，"这一局得了多少分" = 回报。本章剩下全部内容都在回答同一个问题："往上推/往下压"的力度，到底该用什么数字来算才最靠谱？

一个母公式 + 五代进化

整章其实只有一个母公式，剩下全是它的变种。先看核心：

∇θ J(θ) = E[ Σₜ Φₜ · ∇θ log πθ(Aₜ|Sₜ) ]

人话版：想让策略变好，就对每一步做一件事——把"做了当时那个动作的概率"往上抬。抬多用力，由前面的权重 Φₜ说了算。本章 5 个算法的唯一区别，就是 Φₜ 用什么。

版本Φₜ解决了什么毛病
1. 原始策略梯度G(τ)— 起点
2. REINFORCEGₜ别把"做动作前就已有的分"算进来（噪声）
3. 带基线Gₜ − b(Sₜ)全是正分时也能区分"比平均好"还是"比平均差"
4. Actor-CriticRₜ+γV(Sₜ₊₁)−V(Sₜ)不用等整局结束，走一步就能学
5. GAE多步 TD 加权平均在"偏差"和"抖动"之间设一个旋钮

第 1 代：原始策略梯度

Φₜ = G(τ)，这一局所有动作共用"整局总分"同一个权重。能用，但毛病明显：如果某局总分特别高（G=10000），更新步子大到把策略一把拽崩——这叫策略崩溃。

第 2 代：REINFORCE — "别把过去的分算进来"

毛病：第 t 步动作的权重 G(τ) 包含了整局的分，连"做这步之前就已经拿到的分"也算进去了。可是——你这一步不可能改变已经发生的过去。把过去的分掺进来纯属噪声。

修法：第 t 步只用"从这步往后"的回报，记作 Gₜ（也叫 reward-to-go）：

Φₜ = Gₜ = Rₜ + γRₜ₊₁ + ... + γᵀ⁻ᵗRᵀ

人话：评价"第 37 步走得好不好"，只看第 37 步之后赚了多少，之前的账一笔勾销。不影响正确性（无偏），但方差更小——训练更稳更快。

第 3 代：带基线 — "给分数找个及格线"

毛病：倒立摆里每步奖励都是 +1，所以 Gₜ 永远是正的。这意味着不管动作好坏，都在抬高概率，只是抬多抬少的区别。这很蠢——差动作也该被压下去。

修法：减去基线 b(Sₜ)，把"绝对分"变成"相对分"：

Φₜ = Gₜ − b(Sₜ)

别看你考了多少分，看你比平均水平高还是低。比平均好→抬概率；比平均差→压概率。实践中最常用的是价值函数 V(Sₜ) 做基线。"杆已经歪到必倒的状态，怎么推 3 步后都得倒。Gₜ=3，没基线还会傻傻抬概率；但价值函数早看穿了这状态就值 3 分，权重为 0——不瞎调。"

第 4 代：Actor-Critic — "边玩边学"

毛病：前面要等一整局玩完才知道 Gₜ。慢。

修法：用时序差分（TD）——只往前看一步，用价值网络预测"补全"未来：

Φₜ = Rₜ + γV(Sₜ₊₁) − V(Sₜ) = δₜ（1步TD误差）

这里出现两张网：Actor（演员）= 策略网络——负责做动作；Critic（评论家）= 价值网络——负责打分点评。两张网同时训练：评论家努力让自己的估值逼近"实际看到的"，演员听评论家的点评来调整自己。

机器学习类比Actor-Critic ↔ GAN 生成器-判别器
Actor（演员）策略网络，负责"产出"动作
↔
GAN 生成器负责"造假"生成数据
Critic（评论家）价值网络，负责"评估"动作
↔
GAN 判别器负责"鉴别"真假数据

第 5 代：GAE — "偏差和抖动的调节旋钮"

毛病：第 4 代只看 1 步——估得稳但偏；REINFORCE 看到底——没偏但抖。能不能要个中间档？

GAE（广义优势估计）：把"看 1 步、2 步、3 步……"的结果做指数加权平均，用 λ（0~1）当旋钮：

Aₜ^GAE = Σₗ (γλ)ˡ · δₜ₊ₗ
递推：Aₜ^GAE = δₜ + γλ · Aₜ₊₁^GAE

λ=0 → 只看 1 步（稳但偏）；λ=1 → 看到底（准但抖）；典型值 0.95。PPO 里真正在跑的就是这个递推公式——从最后一步往前倒着扫一遍。

机器学习类比策略梯度 ↔ 梯度下降，奖励 ↔ 负损失
奖励 R（越大越好）环境的即时反馈
↔
−损失 L（越大越好）预测误差的负值
策略梯度 ∇log π(a|s)增大好动作的概率
↔
梯度下降 ∇θ L减小预测误差
类比要点：监督学习有 ground truth → 算交叉熵 → 反向传播。RL 没有 ground truth → 采样轨迹 → 用回报做权重 → 梯度上升。优化机制本质相同，区别仅在梯度的来源。

def update(self, state, action_prob, reward, next_state, done):
    # ① 评论家：让 V(Sₜ) 逼近 "Rₜ + γ·V(Sₜ₊₁)"
    target = reward + self.gamma * self.v(next_state) * (1 - done)
    v = self.v(state)
    loss_v = F.mse_loss(v, target.detach())

    # ② 演员：用 TD 误差 δ 当权重
    delta = (target - v).detach()  # δ = Rₜ+γV(Sₜ₊₁)−V(Sₜ)
    loss_pi = -torch.log(action_prob) * delta

    self.optimizer_v.zero_grad(); loss_v.backward();  self.optimizer_v.step()
    self.optimizer_pi.zero_grad(); loss_pi.backward(); self.optimizer_pi.step()

🧠 产品经理视角

三个关键洞察

1. "方差 vs 偏差"是贯穿全章的产品权衡。抖（方差大）→ 训练慢不稳烧算力；偏（偏差大）→ 可能学歪收敛到次优。GAE 的 λ 就是把这个权衡做成可调旋钮——这思路在很多工程系统里通用。

2. "策略崩溃"是真实风险。一次异常大的奖励就能把策略拽崩。所以下一章 PPO 专门加了"步子别迈太大"的护栏——生产级 RL 几乎不用裸策略梯度。

3. 这一章是看懂 RLHF / PPO / GRPO 的钥匙。ChatGPT 做对齐时的强化学习骨架就是本章母公式：让模型"多做能拿高分（人类更喜欢）的回答"。区别只在 Φₜ 怎么算、怎么防崩溃。

PPO — 近端策略优化

🎯 导读

策略梯度法有个要命的毛病——步子一大就学崩，崩了还要花很久才能爬回来。PPO 用一个叫裁剪（clip）的小机关，把更新幅度锁死在安全范围内。这是目前最广泛使用的 RL 算法，也是 OpenAI 的 InstructGPT 和 ChatGPT 的核心训练技术。

🌰 直觉：走钢丝的杂技演员

想象教刚会站立的小孩走路——迈太小磨蹭半天不动，迈太大啪地摔个大跟头。策略梯度法就像一个不看路况的大人，每步按固定力气推。

而走钢丝的杂技演员给了我们另一种思路：他从不大跨步，每一步都先用脚尖探一探，确认脚下安全才挪重心——幅度被牢牢限制在一个小圈里。

PPO 干的就是这件事：不管前面地形多险，每次更新都只准在"上一版策略附近的一个小圈圈里"挪动。这个"小圈圈"就是它名字里"近端（Proximal，离原地不远）"的来历。

为什么策略梯度这么容易崩？

这里藏着一个深度学习里没有、但强化学习里特别毒的问题：

深度学习的训练数据是死的，强化学习的训练数据是活的

训练图像分类器，不管参数怎么变，照片永远摆在那不动。但 RL 里，训练数据（轨迹）是你自己当前策略"玩"出来的。策略一变，下一轮采集到的数据也跟着变。形成恶性循环：步子大→策略变差→拿差策略玩出烂数据→拿烂数据继续训→越训越差→满盘皆输。

PPO 三件套：比值 · 优势 · 裁剪

① 优势 At — "比平均好多少"

评论家给出的关键数字。Aₜ > 0：这步比平均好→鼓励；Aₜ < 0：比平均差→打压。通常用 GAE 计算。

② 比值 ρ — "新策略和旧策略的差异表"

ρₜ = πθ(aₜ|sₜ) / πθ_old(aₜ|sₜ)。ρ=1 新旧一样；ρ=1.5 新策略更爱这个动作；ρ=0.5 没那么爱了。PPO 的整套机制就是盯着 ρ，不准它离 1 太远。

③ 裁剪 — "限位器"

给 ρ 设上下限 [1−ε, 1+ε]（ε=0.2，即 [0.8, 1.2]）。比值想冲 1.5？按 1.2 算。想跌 0.3？按 0.8 算。一旦 ρ 被摁在边界上，梯度变成 0——模型在那个方向"使不上劲"，自然不会再往外冲。

PPO 替代目标函数

J^PPO(θ) = E[ min( ρ·A, clip(ρ, 1−ε, 1+ε)·A ) ]
其中 ρ = πθ(a|s) / πθ_old(a|s)

min 是精髓：保证模型永远拿不到"迈大步"的好处。一旦你想靠"猛增某动作概率"多捞分，min 会把你的收益按裁剪后的天花板掐住，多冲也是白冲。等于告诉模型："小步走有奖，大步走没奖。"

📐 6 种情况分类讨论（可展开）
情况ρA有梯度？
1在 [0.8,1.2]+✅ 正常更新
2在 [0.8,1.2]−✅ 正常更新
3<0.8+✅ 朝范围内拉
4<0.8−❌ 梯度=0
5>1.2+❌ 梯度=0（最反直觉）
6>1.2−✅ 朝范围内拉

情况 5 最反直觉：ρ > 1.2（新策略已经大大增加了这个动作），优势又正——朴素策略梯度还想继续加码。但 PPO 说："你已经够贪了，别再冲了"，梯度=0。这正是"防止迈大步"的体现。

💻 PPO 一轮的主干

epsilon = 0.2
traj = run_policy(pi_old)               # 1) 用旧策略玩一局，存下轨迹
advantages = estimate_advantage(traj)    # 2) 评论家算优势 Aₜ（通常用 GAE）

pi_new = clone(pi_old)                  # 3) 新策略从旧策略出发
for i in range(N):                   # 同一批数据反复更新 N 次
    for (s, a, A) in zip(states, actions, advantages):
        ratio = pi_new.prob(a, s) / pi_old.prob(a, s)
        clipped = clip(ratio, 1 - epsilon, 1 + epsilon)
        obj = min(ratio * A, clipped * A)  # 拿不到"迈大步"的好处
    obj.backward(); optimizer.step()
pi_old = pi_new
PPO 的地位

PPO 是当今 RLHF 的"默认引擎"。ChatGPT、Claude 做对齐时背后调参数的主力算法就是 PPO。理由不是效果最炸，而是最省心——稳定、好调、不容易莫名其妙崩盘。工程上，"不会半夜训崩"比"理论上更优"值钱得多。

🧠 产品经理视角

三个关键洞察

1. "近端"思想到处能借用。任何基于反馈反复迭代的系统——A/B 测试、推荐策略灰度上线——都该警惕"一次改太猛"。灰度发布、限制单次改动幅度，都是工程版的"裁剪"。

2. ε（0.2）是你能拧的旋钮。调小→更稳但更慢（步子更碎）；调大→更快但更易翻车。和上一章学习率的"太大崩太小慢"是同一回事。

3. PPO 不保证每一步都严格变好。它只是个"软约束"，偶尔还是会做错决策。但它胜在"实现简单 + 效果稳 + 数据可复用"，综合性价比依然最高。

PPO 背后的数学

🎯 导读

上一章讲了 PPO "怎么用"——clip 限位器一装，策略更新就不容易崩。但这章要回答一个更底层的问题：为什么 clip 能保证策略变好而不是变差？这需要从保守策略迭代（CPI）的信赖域理论出发，一步步推导出性能下界，最后用 TV 距离和 Pinsker 不等式搭桥到 KL 散度——整条逻辑链就是 TRPO 和 PPO 的理论根基。

数学脉络总览

CPI 替代目标 → 性能下界(误差项含 TV²) → Pinsker: TV ≤ √(KL/2)
    → TRPO: 带 KL 约束的优化 → PPO: clip 做"软约束"近似

KL 散度 — 两个策略的"距离"

衡量两个概率分布的差异（"从 p 的角度看，q 有多奇怪"）：

D_KL(p‖q) = Σ p(x) log [p(x)/q(x)]

两大性质：≥0（只当两分布相同时为 0）；非对称——D_KL(p‖q) ≠ D_KL(q‖p)。为什么非对称很重要？在 RLHF 中，我们要求微调后的模型不偏离参考模型太远——方向性很重要。

为什么必须有 KL 惩罚？

奖励黑客（Reward Hacking）：模型利用奖励模型的漏洞获得高分但生成无意义文本（比如发现奖励模型偏爱长回答，就疯狂输出废话）。KL 惩罚拽住模型，让它仍然是"人话"——没有 KL，模型会生成乱码骗高分。

机器学习类比KL 散度惩罚 ↔ L1/L2 正则化
KL 惩罚 β·D_KL[πθ‖πref]防止策略偏离参考模型
↔
L1/L2 正则化 λ·||w||²防止权重过大过拟合
类比要点：都是加在目标函数上的"约束项"。正则化防过拟合，KL 防奖励黑客。超参数 β 和 λ 也对应：太大欠拟合，太小过拟合。

重要性采样 — 用旧数据更新新策略

允许用旧策略采样的数据来更新新策略，通过权重修正偏差：

E_{x~p}[f(x)] = E_{x~q}[ f(x) · p(x)/q(x) ]

但如果 p(x)/q(x) 在某些位置很大，方差会爆炸——所以 PPO 用 clip 限制这个比值。

详细推导见重要性采样。

信赖域理论 — PPO 为什么能保证策略改进

PPO 的 clip 不是拍脑袋设计的——它背后有一套严格的理论保证。核心逻辑链：

1. 保守策略迭代(CPI)：J(π') − J(π) ≥ L_π(π') − C·D_TV(π'‖π)²
2. Pinsker不等式：D_TV ≤ √(KL/2)，将TV转为KL
3. TRPO：显式约束 KL ≤ δ，共轭梯度求解
4. PPO：用 clip 软化约束，实现简单但效果等价

完整推导链见公式详解页面：

  • 保守策略迭代 (CPI) 完整推导 — 从 Performance Difference Lemma 到替代目标
  • 信赖域与 PPO 理论保证 — TRPO → PPO 的工程简化逻辑
  • TV 距离与 Pinsker 不等式 — 从 TV 到 KL 的数学桥梁
  • 重要性采样方差分析 — 为什么 clip 能防方差爆炸

GRPO — 组相对策略优化

🎯 导读

PPO 又强又稳，但它得额外养一个"评论家"（价值网络）来打分——又贵又难训。GRPO（Group Relative Policy Optimization，说人话就是"让一组答案互相比，比平均好就鼓励、差就压制"）干脆把评论家炒了，省一半算力。这正是 DeepSeek-R1 训练时用的招。

🌰 直觉：老师改卷

回忆 PPO：做了个动作，得知道"这一步是好是坏"。但好坏是相对的——考 60 分是好是坏？得看平均分。所以 PPO 养了一个评论家实时估计"这局面一般能拿多少分"当基线。

GRPO 的灵感特别朴素：

"一道大题，我让同一个模型做 5 遍，得到 5 份答案各自打分，算平均分。比平均分高的那几份——'就该这么写，多来点'；比平均分低的——'这思路不行，少来'。"

看出门道了吗？根本不需要另请评论家——同组里其它答案就是天然的参照线。好坏不看绝对分，只看你比同组小伙伴强还是弱。

GRPO 的优势计算

对每个问题采样 G 条回答，算每条奖励，然后组内归一化：

Aᵢ = (rᵢ − μ) / σ
μ = mean(r₁, ..., rG), σ = std(r₁, ..., rG)

分子 rᵢ−μ：你比平均高多少；除以 σ：按难易程度缩放。高于平均→正优势→被强化；低于平均→负优势→被削弱。用同伴表现做基准，不需要额外训练价值网络。

机器学习类比GRPO 组相对优势 ↔ Batch 内归一化
GRPO A = (r−μ)/σ组内轨迹奖励的归一化
↔
Batch Normalizationbatch 内特征的归一化
不用 Critic，用组均值替代用同伴表现作为基准
↔
不用 running mean，用 batch 均值用当前 batch 统计量替代

GRPO 目标函数

J^GRPO(θ) = (1/G) Σᵢ (1/|τᵢ|) Σₜ min[ρ·A, clip(ρ,1−ε,1+ε)·A] − β·D_KL[πθ‖πref]

和 PPO 的唯一本质区别：PPO 的 A 来自"价值网络+GAE"，GRPO 的 A 来自"组内标准化"。其余框架完全照搬。省掉评论家网络后，显存占用和训练复杂度都明显下降——对几十上百 B 参数的大模型，省下的 GPU 成本是真金白银。

💻 GRPO 核心代码

# 第一步：算组内优势
def calc_advantages(trajectories):
    rewards = [get_return(t) for t in trajectories]
    mean = sum(rewards) / len(rewards)
    std  = np.std(rewards) + 1e-8
    return [(r - mean) / std for r in rewards]

# 第二步：采一组，用 PPO 裁剪框架更新
G = 5  # 一组 5 条轨迹
trajectories = [agent.collect(env) for _ in range(G)]
advantages = calc_advantages(trajectories)
for _ in range(20):  # 同一批数据多更新几次
    loss = sum(min(ratio * A, clip(ratio, 0.8, 1.2) * A) for ...)
    loss.backward(); optimizer.step()
GRPO 特别适合"答案能被批量打分"的场景

数学题（对/错可验证）、代码（能不能跑通）、推理题（有标准答案）。一道题生成一堆答案，按对错打分，组内一比就能学——这正是DeepSeek-R1 练就强推理能力的核心套路。

🧠 产品经理视角

三个关键洞察

1. 卖点就俩字：省钱。砍掉价值网络后，显存和复杂度都明显下降。对于大模型，"少养一个同等规模的评论家"省的是真金白银。

2. 软肋也来自"组相对"。如果一组答案全错或全对，组内没差异（标准差趋近 0），这组就几乎学不到东西。采样的"组大小 G"和题目难度要搭配好。

3. 选型直觉：打分信号稠密且需要精细信用分配 → PPO 有价值；信号稀疏、只在结局给分、追求省成本 → GRPO 更划算。大模型 RLHF 这几年明显在往 GRPO 倒。
